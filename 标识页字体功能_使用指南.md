# 标识页字体功能使用指南

## 功能概述

标识页字体功能允许用户从系统已安装的字体中选择合适的字体，用于生成 PDF 标识页。系统会自动验证字体兼容性，确保预览效果与最终 PDF 生成效果完全一致。

## 使用步骤

### 1. 打开字体设置

1. 启动应用程序
2. 进入"设置"界面
3. 找到"标识页字体"设置区域

### 2. 选择字体

**字体列表特点：**
- 显示系统所有可用字体
- 中文字体（如微软雅黑、黑体、宋体等）优先显示在列表顶部
- 加载速度快，不会卡顿

**操作步骤：**
1. 点击字体下拉列表
2. 浏览可用字体
3. 选择您喜欢的字体

### 3. 查看预览

**预览区域显示：**
```
字体预览: [字体名称] - 中文测试 ABC 123
```

**预览状态（异步更新）：**
- ✅ **黑色文字**：字体完全兼容，可以正常使用
- ⚠️ **红色文字 + 警告**：字体可能不兼容，建议选择其他字体

**注意：** 预览会先显示黑色文字，然后在后台验证 PDFsharp 兼容性。如果不兼容，会自动变为红色并显示警告。

**示例：**
```
✅ 字体预览: Microsoft YaHei - 中文测试 ABC 123
⚠️ 字体预览: SomeFont - 中文测试 ABC 123 ⚠️ 此字体可能无法在PDF中正常显示
```

### 4. 保存设置

1. 确认字体预览效果满意
2. 点击"保存"按钮
3. 设置会自动保存到配置文件

### 5. 使用标识页功能

1. 在文件处理界面勾选"标识页"选项
2. 设置标识页内容（如"测试文档-中文English123"）
3. 执行文件处理
4. 生成的 PDF 第一页将使用您选择的字体显示标识页

## 常见问题

### Q1: 为什么字体列表加载很快？

**A:** 系统采用两阶段验证策略：
1. **加载时**：只做基本的 WinForms 验证，速度很快
2. **选择时**：异步验证 PDFsharp 兼容性，不阻塞界面

这样既保证了加载速度，又能及时发现兼容性问题。

### Q2: 预览显示正常，但生成 PDF 时出错怎么办？

**A:** 这种情况很少见，因为系统会在预览时同时验证 PDFsharp 兼容性。如果出现此问题：

1. 查看日志文件（app_YYYY-MM-DD.log）
2. 尝试选择其他字体
3. 重新加载字体列表（点击"重新加载"按钮）

### Q3: 字体预览显示红色警告怎么办？

**A:** 红色警告表示该字体可能无法在 PDF 中正常显示。建议：

1. 选择其他字体
2. 如果必须使用该字体，请先测试生成小文件
3. 查看生成的 PDF 是否正常显示

### Q4: 如何添加新字体？

**A:** 
1. 在 Windows 系统中安装新字体
2. 重启应用程序或点击"重新加载"按钮
3. 新字体会自动出现在列表中（如果兼容）

### Q5: 标识页文字太长怎么办？

**A:** 系统会自动处理长文本：
- 自动换行
- 垂直居中
- 如果文本过长，会自动截断并添加省略号

## 技术说明

### 字体兼容性验证

系统采用两阶段验证策略：

**阶段1：加载时（快速验证）**
```
1. 遍历系统字体
2. 尝试创建 WinForms 字体对象
3. 成功 → 添加到列表
   失败 → 跳过
```

**阶段2：选择时（异步验证）**
```
1. 用户选择字体
2. 立即显示 WinForms 预览
3. 后台异步测试 PDFsharp 兼容性
4. 验证完成后更新显示
   - 兼容 → 黑色文字
   - 不兼容 → 红色警告
```

### 预览验证

字体预览采用渐进式验证：

```
1. 立即显示 WinForms 预览（快速）
2. 异步验证 PDFsharp 兼容性（后台）
3. 验证完成后更新颜色提示
```

### 所见即所得

- 预览效果 = PDF 生成效果
- 不会出现字体自动替换
- 不会出现乱码或方框
