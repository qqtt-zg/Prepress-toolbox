# 大诚重命名工具

一个功能强大的文件批量重命名工具，专为生产环境设计，支持多种重命名规则、Excel 数据导入、PDF 处理等高级功能。

## 主要功能

### 🔄 批量文件重命名

- 支持多种重命名规则和模式
- 智能识别返单文件（带前缀标识）
- 正则表达式匹配和字段提取
- 预览重命名结果，确保准确性

### 📊 Excel 数据导入

- 支持从 Excel 文件批量导入重命名数据
- 自动数据匹配和字段提取
- 支持列组合功能
- 智能处理未分组项目

### 📋 事件分组管理

- 9 个专业预定义分组（订单、材料、数量、工艺、客户、备注、行数、列数、未分组）
- TreeView 层级化管理
- 支持拖拽排序和项目移动
- 独立前缀配置和启用/禁用控制

### 🏷️ 保留分组冲突检测

- 智能检测保留冲突
- 友好的冲突提示对话框
- 自动清理现有保留状态

### 📄 PDF 处理功能

- PDF 文件预览和处理
- 支持形状识别（直角矩形、圆形、圆角矩形、异形）
- PDF 缩放和导航功能
- 折叠动画效果

#### 已知问题

- **悬浮拖拽窗口拖拽提示文字不显示** ⚠️
  - **现象**: 拖拽文件到悬浮拖拽窗口时，鼠标指针右下角只显示很小的长方形，而不是完整的拖拽提示文字（"复制到..."、"移动到..."）
  - **影响**: 不影响功能使用，但用户体验不够友好
  - **原因**: Windows Forms 的拖拽提示文字（"复制到..."、"移动到..."）通常只在从文件资源管理器拖拽文件时才会显示。从应用程序内部拖拽时，系统可能只显示默认的小长方形反馈
  - **当前实现**: 已正确设置 `DragDropEffects`（Copy/Move），系统会根据效果显示相应的光标，但提示文字可能不显示
  - **待优化**:
    - ⏳ 实现自定义拖拽图像（DragImage）以显示完整的拖拽反馈
    - ⏳ 或使用 Win32 API 实现自定义拖拽提示文字
  - **技术参考**: `src/WindowsFormsApp3/Forms/Utils/FloatingDropZoneForm.cs` 中的 `OnDragEnter` 和 `OnDragOver` 方法

- **PDF 缩放轻微跳动** ⚠️
  - **现象**: 使用 Ctrl+滚轮缩放 PDF 时，画面可能出现轻微的视觉跳动（约 2-5px）
  - **影响**: 不影响正常使用，可接受
  - **原因**: DOM 布局重排和滚动位置同步的时序问题
  - **已实施优化**:
    - ✅ 双重 requestAnimationFrame 确保布局稳定
    - ✅ 防抖优化到 20ms（极速响应）
    - ✅ 移除 CSS transform 预览机制
  - **待优化**:
    - ⏳ 进一步优化滚动位置计算精度
    - ⏳ 探索三重 RAF 或延迟调整方案
  - **临时方案**: 使用"适配宽度"或"适配页面"按钮重置视图
  - **技术参考**: `src/WindowsFormsApp3/Resources/pdfjs/bridge.js:260-285`

## 📖 印前排版术语说明

> **重要**: 以下术语定义遵循国内印刷行业标准，开发时请使用正确的术语。

### 连拼 (Step and Repeat)

将**相同**的设计按固定间距重复排列在版面上，形成网格状布局。

```
┌───┬───┬───┬───┐
│ A │ A │ A │ A │  ← 同一个标签
├───┼───┼───┼───┤
│ A │ A │ A │ A │  ← 重复排列
├───┼───┼───┼───┤
│ A │ A │ A │ A │
└───┴───┴───┴───┘
```

**特点**: 相同内容、固定间距、网格排列  
**应用**: 不干胶标签、名片、吊牌等  
**项目实现**: `ImpositionService` 中的布局计算功能

### 折手 (Signature Imposition)

将**相同尺寸**的页面/标签按**特定顺序**排列，用于后续折叠、装订或拣货时顺序正确呈现。

```
示例1 - 书刊装订：          示例2 - 标签可变数据：
印刷版面：    折叠后：       印刷版面：         拣货顺序：
┌───┬───┐    ┌───┐         ┌───┬───┬───┐      1→2→3→4
│ 4 │ 1 │ →  │ 1 │         │ 1 │ 2 │ 3 │      (按Z字形或
├───┼───┤    │ 2 │         ├───┼───┼───┤       指定路径)
│ 2 │ 3 │    │ 3 │         │ 4 │ 5 │ 6 │
└───┴───┘    │ 4 │         └───┴───┴───┘
             └───┘
```

**特点**: 相同尺寸、考虑最终使用顺序  
**应用**:

- 书刊装订：骑马钉、折页、画册内页
- **标签可变数据**：一式多联标签、流水号标签、配对标签的折叠/拣货顺序排列

**项目状态**: ⏳ 待开发

### 合版 (Gang Run) - 待定义

> ⚠️ **术语预留**: 当需要实现**不同尺寸**的混合拼版功能时，使用"合版"术语。

将**不同设计**或**不同尺寸**的内容拼在同一版面上，优化材料利用。

```
┌─────┬───┬───────┐
│  A  │ B │   C   │  ← 不同尺寸
├─────┼───┴───┬───┤
│  D  │   E   │ F │  ← 混合拼版
└─────┴───────┴───┘
```

**特点**: 不同内容、不同尺寸、灵活布局  
**应用**: 合版印刷、混合订单  
**项目状态**: 🔮 未规划

## 使用指南

### 基本重命名流程

1. 将需要重命名的文件添加到程序中
2. 设置重命名规则（支持正则表达式）
3. 预览重命名结果
4. 确认执行批量重命名

### Excel 数据导入

1. 准备包含重命名数据的 Excel 文件
2. 点击"导入 Excel"按钮
3. 选择 Excel 文件和对应的工作表
4. 配置列映射关系
5. 导入数据并应用重命名

### 事件分组管理

1. 在事件管理界面中创建或编辑分组
2. 将相关项目拖拽到对应分组
3. 配置分组前缀和规则
4. 启用或禁用特定分组

### PDF 处理

1. 选择需要处理的 PDF 文件
2. 使用 PDF 预览功能查看内容
3. 选择识别的形状类型
4. 应用处理规则
