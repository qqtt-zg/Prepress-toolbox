# 转曲工作流测试指南

## 快速测试步骤

### 测试1: 转曲 → 保存（覆盖原文件）
1. 启动应用程序
2. 打开一个PDF文件（建议使用测试文件）
3. 点击工具栏的"检查器"按钮
4. 在检查器窗口中，切换到"字体"标签页
5. 点击"转曲"按钮
6. 在确认对话框中点击"确定"
7. 等待转曲完成（应该看到成功提示）
8. **验证点**: 
   - 预览窗口应该自动刷新显示转曲后的PDF
   - 状态栏显示"已转曲 (未保存)"
   - "保存"按钮应该变为可用状态
9. 点击工具栏的"保存"按钮
10. **验证点**:
    - 应该看到"文件已保存"提示
    - 状态栏更新为"已保存: 文件名.pdf"
    - 原文件已被覆盖（可以用其他PDF阅读器打开验证）

### 测试2: 转曲 → 另存为（保存到新文件）
1. 启动应用程序
2. 打开一个PDF文件
3. 点击"检查器"按钮
4. 在检查器的"字体"标签页点击"转曲"按钮
5. 确认转曲操作
6. 等待转曲完成
7. **验证点**: 
   - 预览显示转曲后的效果
   - "保存"和"另存为"按钮都可用
8. 点击工具栏的"另存为"按钮
9. 在保存对话框中：
   - **验证点**: 默认文件名应该是"原文件名_转曲.pdf"
   - 选择保存位置
   - 点击"保存"
10. 在"是否打开新文件"对话框中点击"打开"
11. **验证点**:
    - 新文件已创建
    - 预览窗口显示新文件
    - 状态栏显示新文件名
    - 原文件未被修改

### 测试3: 转曲 → 不保存 → 打开新文件
1. 打开一个PDF文件
2. 执行转曲操作
3. **不要保存**
4. 直接点击"打开PDF"按钮打开另一个文件
5. **验证点**:
   - 新文件正常加载
   - 状态重置（"保存"按钮禁用）
   - 之前的转曲结果丢失（这是预期行为）

### 测试4: 按钮状态验证
1. 启动应用程序
2. **验证点**: 所有按钮都应该是禁用状态（除了"打开PDF"）
3. 打开一个PDF文件
4. **验证点**: 
   - "另存为"按钮启用
   - "保存"按钮仍然禁用（因为没有更改）
   - 其他功能按钮启用
5. 执行转曲操作
6. **验证点**:
   - "保存"按钮启用
   - "另存为"按钮启用
7. 点击"保存"
8. **验证点**:
   - "保存"按钮再次禁用（因为已保存）
   - "另存为"按钮仍然启用

### 测试5: 错误处理
1. 打开一个不包含文字的PDF（纯图片PDF）
2. 执行转曲操作
3. **验证点**: 应该能正常完成（即使没有文字可转曲）

## 常见问题排查

### 问题1: 转曲按钮不可见或禁用
- 检查是否已加载PDF文件
- 检查检查器窗口是否正确打开
- 查看日志文件确认按钮创建是否成功

### 问题2: 转曲后预览没有更新
- 检查临时文件是否创建成功
- 查看日志确认事件是否正确触发
- 验证 `Inspector_FontOutlineCompleted` 方法是否被调用

### 问题3: 保存按钮始终禁用
- 检查 `_hasUnsavedChanges` 标志是否正确设置
- 验证 `FontOutlineCompleted` 事件是否正确订阅
- 查看日志确认事件处理流程

### 问题4: 另存为后原文件被修改
- 这不应该发生，如果发生请报告bug
- 检查 `BtnSaveAs_Click` 方法的实现

## 日志检查

转曲操作会产生以下日志：
```
[PdfInspectorControl] 收到转曲按钮点击
[字体转曲] 开始处理: 文件名.pdf
[字体转曲] 使用 Ghostscript: 路径
[字体转曲] 完成: 输出文件名.pdf
[PdfOperationsPanel] 收到字体转曲完成事件
[PdfOperationsPanel] 转曲后的PDF已加载到预览
```

如果出现问题，请检查日志文件：
- `app_2026-01-19.log` (当天日志)
- 查找包含 "[字体转曲]" 或 "[PdfOperationsPanel]" 的行

## 性能测试

建议测试不同大小的PDF文件：
- 小文件 (< 1MB): 应该几秒内完成
- 中等文件 (1-10MB): 可能需要10-30秒
- 大文件 (> 10MB): 可能需要更长时间

注意：转曲时间主要取决于：
1. PDF页数
2. 文字数量
3. 字体复杂度
4. Ghostscript性能
