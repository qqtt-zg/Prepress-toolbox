# PDF操作面板 - 撤回/重做系统快速参考

## 用户操作指南

### 基本操作

#### 撤回操作
- **按钮**: 点击工具栏"撤回 (X)"按钮
- **快捷键**: `Ctrl+Z`
- **功能**: 撤回最近的字体转曲操作
- **显示**: 按钮显示可撤回的操作数量

#### 重做操作
- **按钮**: 点击工具栏"重做 (X)"按钮
- **快捷键**: `Ctrl+Y` 或 `Ctrl+Shift+Z`
- **功能**: 恢复被撤回的操作
- **显示**: 按钮显示可重做的操作数量

### 工作流程

```
打开PDF → 转曲(1) → 转曲(2) → 转曲(3)
                                    ↓
                            撤回 (3) → (2)
                                    ↓
                            撤回 (2) → (1)
                                    ↓
                            重做 (1) → (2)
                                    ↓
                            转曲(4) → 重做栈清空
```

### 按钮状态说明

| 状态 | 撤回按钮 | 重做按钮 | 保存按钮 |
|------|---------|---------|---------|
| 刚打开文件 | 禁用 | 禁用 | 禁用 |
| 转曲后 | 撤回 (1) | 禁用 | 启用 |
| 撤回后 | 撤回 (0) 或数字 | 重做 (1) | 启用 |
| 重做后 | 撤回 (1) 或数字 | 重做 (0) 或数字 | 启用 |
| 保存后 | 禁用 | 禁用 | 禁用 |

## 开发者参考

### 核心数据结构

```csharp
// 历史栈
private Stack<byte[]> _undoStack = new Stack<byte[]>();  // 撤回栈
private Stack<byte[]> _redoStack = new Stack<byte[]>();  // 重做栈

// PDF状态
private byte[] _originalPdfBytes;  // 原始PDF
private byte[] _currentPdfBytes;   // 当前显示的PDF

// 配置
private const int MAX_HISTORY_SIZE = 10;  // 最大历史数
```

### 关键方法

#### 1. 加载文件
```csharp
private async Task LoadPdfFileAsync(string filePath)
{
    _originalPdfBytes = File.ReadAllBytes(filePath);
    _currentPdfBytes = _originalPdfBytes;
    _undoStack.Clear();
    _redoStack.Clear();
    UpdateHistoryButtons();
}
```

#### 2. 转曲完成
```csharp
private void Inspector_FontOutlineCompleted(object sender, FontOutlineCompletedEventArgs e)
{
    _undoStack.Push(_currentPdfBytes);  // 保存当前状态
    _currentPdfBytes = e.PdfBytes;      // 更新为新状态
    _redoStack.Clear();                 // 清除重做栈
    LimitHistorySize();                 // 限制栈大小
    UpdateHistoryButtons();
}
```

#### 3. 撤回操作
```csharp
private void BtnUndo_Click(object sender, EventArgs e)
{
    _redoStack.Push(_currentPdfBytes);      // 当前状态入重做栈
    _currentPdfBytes = _undoStack.Pop();    // 弹出上一状态
    UpdateHistoryButtons();
    ShowPdfPreview(_currentPdfBytes);
}
```

#### 4. 重做操作
```csharp
private void BtnRedo_Click(object sender, EventArgs e)
{
    _undoStack.Push(_currentPdfBytes);      // 当前状态入撤回栈
    _currentPdfBytes = _redoStack.Pop();    // 弹出重做状态
    UpdateHistoryButtons();
    ShowPdfPreview(_currentPdfBytes);
}
```

#### 5. 更新按钮
```csharp
private void UpdateHistoryButtons()
{
    _btnUndo.Enabled = _undoStack.Count > 0;
    _btnRedo.Enabled = _redoStack.Count > 0;
    
    _btnUndo.Text = _undoStack.Count > 0 
        ? $"撤回 ({_undoStack.Count})" 
        : "撤回";
    _btnRedo.Text = _redoStack.Count > 0 
        ? $"重做 ({_redoStack.Count})" 
        : "重做";
}
```

#### 6. 限制历史大小
```csharp
private void LimitHistorySize()
{
    if (_undoStack.Count > MAX_HISTORY_SIZE)
    {
        var items = _undoStack.ToList();
        items.RemoveAt(items.Count - 1);  // 移除最旧的
        _undoStack = new Stack<byte[]>(items.Reverse());
    }
}
```

### 状态转换图

```
[原始状态]
    ↓ 转曲
[状态1] ← 撤回栈
    ↓ 转曲
[状态2] ← 撤回栈
    ↓ 转曲
[状态3] ← 当前状态
    ↓ 撤回
[状态2] ← 当前状态, [状态3] → 重做栈
    ↓ 撤回
[状态1] ← 当前状态, [状态2,状态3] → 重做栈
    ↓ 重做
[状态2] ← 当前状态, [状态3] → 重做栈
    ↓ 转曲（新操作）
[状态4] ← 当前状态, 重做栈清空
```

## 技术细节

### 内存管理

#### 内存占用估算
```
总内存 = 原始PDF大小 + (撤回栈大小 × PDF大小) + (重做栈大小 × PDF大小)

示例：
- PDF文件: 5MB
- 撤回栈: 3个状态
- 重做栈: 2个状态
- 总内存: 5MB + (3 × 5MB) + (2 × 5MB) = 30MB
```

#### 优化策略
1. **限制历史大小**: 最多10个状态
2. **移除最旧记录**: 超过限制时自动清理
3. **保存时清除**: 保存后清空所有历史

### 性能考虑

#### 操作复杂度
- 撤回: O(1) - 栈弹出操作
- 重做: O(1) - 栈弹出操作
- 限制大小: O(n) - n为历史数量
- 预览显示: O(1) - 写临时文件

#### 响应时间
- 撤回/重做: < 100ms（小文件）
- 预览加载: < 500ms（取决于PDF大小）
- 历史限制: < 50ms

## 常见问题

### Q1: 为什么保存后历史被清除？
**A**: 保存后文件已经持久化，历史记录不再需要。这是标准行为，与 Adobe Acrobat 一致。

### Q2: 可以增加历史记录数量吗？
**A**: 可以修改 `MAX_HISTORY_SIZE` 常量，但要注意内存占用。

### Q3: 为什么新操作会清除重做栈？
**A**: 这是标准的撤回/重做逻辑。新操作创建了新的历史分支，旧的重做路径不再有效。

### Q4: 支持其他操作的撤回吗？
**A**: 当前只支持字体转曲。未来可以扩展到裁切标记、套准标记等操作。

### Q5: 键盘快捷键不工作？
**A**: 确保焦点在 PdfOperationsPanel 上。快捷键通过 `ProcessCmdKey` 实现。

## 扩展建议

### 未来可能的改进

1. **历史面板**
   - 显示所有操作历史
   - 点击跳转到特定状态
   - 显示操作时间和描述

2. **多种操作类型**
   - 裁切标记
   - 套准标记
   - 页面旋转
   - 页面删除

3. **保存后保留历史**
   - 保存时不清除历史

   - 保存历史到文件
   - 关闭后重新打开可恢复
   - 支持会话恢复

```csharp
// 1. 在执行操作前保存当前状态
_undoStack.Push(_currentPdfBytes);

byte[] newPdfBytes = PerformSomeOperation(_currentPdfBytes);
_currentPdfBytes = newPdfBytes;

// 3. 清除重做栈
_redoStack.Clear();

// 4. 限制历史大小
LimitHistorySize();

// 5. 更新UI
```

```csharp
// 创建历史项类
public class HistoryItem
{
    public byte[] PdfBytes { get; set; }
    public string Description { get; set; }
    public DateTime Timestamp { get; set; }
}

// 使用历史项栈
private Stack<HistoryItem> _undoStack = new Stack<HistoryItem>();

// 添加操作
_undoStack.Push(new HistoryItem
{

    Description = "字体转曲",
    Timestamp = DateTime.Now
});
```
