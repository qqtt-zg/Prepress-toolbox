# 字体预览功能说明

## 功能特点

### 可输入测试文本框

用户可以在预览框中输入任意文本来测试字体效果，不再局限于固定的预览文本。

### 实时字体应用

- 选择字体后，预览框立即应用该字体
- 用户输入文本时，保持当前选择的字体
- 支持中英文混合、数字、符号等

### 兼容性状态提示

预览框上方的提示标签会显示字体兼容性状态（使用实际PDF生成API测试）：

| 状态 | 颜色 | 说明 |
|------|------|------|
| ✓ PDF兼容 | 绿色 | 字体完全兼容，可以正常使用 |
| ✗ PDF不兼容 | 红色 | 字体不兼容，无法生成PDF，请选择其他字体 |
| 验证中... | 灰色 | 正在使用实际PDF API验证兼容性 |

**重要说明：**
- 验证使用与实际标识页生成**完全相同**的 PDFsharp API
- 测试过程：创建真实PDF文档 → 添加页面 → 使用选择的字体绘制文本 → 测量文本宽度
- 结果是**确定性的**，不存在"可能不兼容"的情况
- 如果显示 ✓ 绿色，保证在实际PDF生成时可以正常使用
- 如果显示 ✗ 红色，说明该字体无法在PDF中使用，必须选择其他字体

## 使用方法

### 1. 选择字体
从下拉列表中选择字体

### 2. 查看默认预览
预览框会显示默认文本：`字体预览: 中文测试 ABC 123`

### 3. 输入自定义文本
- 点击预览框
- 清空或修改文本
- 输入您想测试的内容

### 4. 查看兼容性状态
- 提示标签会在1-2秒后显示验证结果
- 绿色 ✓ = 可以使用
- 红色 ⚠️ = 建议更换

## 技术细节

### 字体应用
- 使用 WinForms Font 对象
- 字体大小: 14pt
- 支持多行显示

### 兼容性验证（确定性结果）
- 使用实际的 PDFsharp API 测试
- 创建真实的 PDF 文档和页面
- 使用选择的字体绘制测试文本
- 测量文本宽度验证渲染成功
- 异步验证，不阻塞UI
- 1-2秒后显示确定性结果

**验证代码与实际标识页生成使用相同的 API：**
```csharp
private bool TestPdfSharpFontRendering(string fontName)
{
    try
    {
        // 使用实际的 PDF 生成 API 测试字体
        using (var testDoc = new PdfDocument())
        {
            var testPage = testDoc.AddPage();
            testPage.Width = XUnit.FromPoint(200);
            testPage.Height = XUnit.FromPoint(100);

            using (var gfx = XGraphics.FromPdfPage(testPage))
            {
                // 创建字体（与实际PDF生成使用相同的方式）
                var testFont = new XFont(fontName, 12, XFontStyleEx.Regular);
                
                // 测试绘制用户输入的文本（或默认测试文本）
                string testText = txtFontPreview.Text;
                if (string.IsNullOrEmpty(testText))
                {
                    testText = "中文测试ABC123";
                }

                // 使用与实际标识页相同的绘制方式
                gfx.DrawRectangle(XBrushes.White, 0, 0, 200, 100);
                gfx.DrawString(testText, testFont, XBrushes.Black, 
                    new XPoint(10, 50));
                
                // 测量文本宽度，确保字体能正确渲染
                var size = gfx.MeasureString(testText, testFont);
                
                // 如果能成功绘制且测量到有效宽度，说明字体兼容
                bool isCompatible = size.Width > 0;
                
                LogHelper.Debug($"字体 {fontName} 测试结果: {(isCompatible ? "兼容" : "不兼容")}, 文本宽度: {size.Width}");
                return isCompatible;
            }
        }
    }
    catch (Exception ex)
    {
        LogHelper.Debug($"PDFsharp 字体测试失败 [{fontName}]: {ex.Message}");
        return false; // 任何异常都表示不兼容
    }
}
```

**测试逻辑说明：**
1. 创建真实的 PdfDocument 对象（不是模拟）
2. 添加真实的 PDF 页面
3. 使用 XGraphics 绘制图形（与实际标识页生成相同）
4. 创建 XFont 字体对象（与实际标识页生成相同）
5. 绘制用户输入的测试文本
6. 测量文本宽度验证渲染成功
7. 如果任何步骤失败，返回 false（不兼容）
8. 如果全部成功且宽度 > 0，返回 true（兼容）

**为什么结果是确定性的：**
- 使用的是实际的 PDFsharp API，不是近似测试
- 测试代码路径与生产代码路径完全一致
- 如果测试通过，实际生成时必然通过
- 如果测试失败，实际生成时必然失败
- 不存在"可能"的情况，只有"能"或"不能"

## 常见问题

### Q: 为什么输入文本后字体变了？
A: 这是正常的，说明当前选择的字体不支持您输入的某些字符。建议选择其他字体。

### Q: 预览正常但提示不兼容？
A: WinForms 预览和 PDF 生成使用不同的渲染引擎。建议选择提示兼容的字体。

### Q: 可以输入多长的文本？
A: 预览框支持多行文本，但实际 PDF 标识页会根据页面大小自动换行和截断。

### Q: 如何恢复默认预览文本？
A: 重新选择字体即可恢复默认文本。

## 最佳实践

1. **先测试常用内容**：输入实际会用到的标识页内容
2. **关注兼容性提示**：优先选择绿色 ✓ 的字体
3. **测试特殊字符**：如果需要使用特殊符号，提前测试
4. **保存测试结果**：记录哪些字体效果最好
