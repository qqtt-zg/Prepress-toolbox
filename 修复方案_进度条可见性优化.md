# 进度条可见性优化 - 修复方案

## 修复目标
1. 添加详细的日志记录，追踪进度条行为
2. 增加最小显示时间，确保进度条可见
3. 优化UI更新，避免被跳过

## 修复代码

### 1. 添加字段（在类的开头）

```csharp
// 进度条相关字段
private DateTime _progressStartTime;
private const int MIN_PROGRESS_DISPLAY_TIME_MS = 300; // 最小显示时间：300ms
```

### 2. 修改 ShowBottomProgress 方法

```csharp
/// <summary>
/// 显示底部进度条
/// </summary>
private void ShowBottomProgress()
{
    if (InvokeRequired)
    {
        Invoke(new Action(ShowBottomProgress));
        return;
    }
    
    _progressBar.Visible = true;
    _progressBar.Width = 0;
    _progressStartTime = DateTime.Now;
    
    // 强制刷新UI
    _progressBar.Refresh();
    statusPanel.Refresh();
    
    LogHelper.Debug($"[进度条] 显示进度条，初始宽度: 0");
}
```

### 3. 修改 UpdateBottomProgress 方法

```csharp
/// <summary>
/// 更新底部进度条
/// </summary>
private void UpdateBottomProgress(int percent)
{
    if (InvokeRequired)
    {
        Invoke(new Action<int>(UpdateBottomProgress), percent);
        return;
    }
    
    int targetWidth = (int)((statusPanel.Width - 20) * percent / 100.0); // 减去padding
    _progressBar.Width = targetWidth;
    
    // 强制刷新UI
    _progressBar.Refresh();
    
    LogHelper.Debug($"[进度条] 更新进度: {percent}%, 宽度: {targetWidth}px");
}
```

### 4. 修改 HideBottomProgress 方法

```csharp
/// <summary>
/// 隐藏底部进度条
/// </summary>
private async void HideBottomProgress()
{
    if (InvokeRequired)
    {
        Invoke(new Action(HideBottomProgress));
        return;
    }
    
    // 计算已显示时长
    var displayTime = (DateTime.Now - _progressStartTime).TotalMilliseconds;
    LogHelper.Debug($"[进度条] 准备隐藏，已显示: {displayTime:F0}ms");
    
    // 如果显示时间不足最小时间，等待
    if (displayTime < MIN_PROGRESS_DISPLAY_TIME_MS)
    {
        var remainingTime = (int)(MIN_PROGRESS_DISPLAY_TIME_MS - displayTime);
        LogHelper.Debug($"[进度条] 等待 {remainingTime}ms 以达到最小显示时间");
        await Task.Delay(remainingTime);
    }
    
    _progressBar.Visible = false;
    _progressBar.Width = 0;
    
    // 强制刷新UI
    _progressBar.Refresh();
    statusPanel.Refresh();
    
    LogHelper.Debug($"[进度条] 已隐藏，总显示时长: {(DateTime.Now - _progressStartTime).TotalMilliseconds:F0}ms");
}
```

## 进一步优化建议

### 1. 添加平滑动画
使用 Timer 实现宽度的平滑过渡：
```csharp
private void AnimateProgressBar(int targetWidth)
{
    var timer = new Timer();
    timer.Interval = 10;
    int currentWidth = _progressBar.Width;
    int step = (targetWidth - currentWidth) / 10;
    
    timer.Tick += (s, e) =>
    {
        if (Math.Abs(_progressBar.Width - targetWidth) < Math.Abs(step))
        {
            _progressBar.Width = targetWidth;
            timer.Stop();
        }
        else
        {
            _progressBar.Width += step;
        }
    };
    
    timer.Start();
}
```

## 注意事项

1. **线程安全**：所有UI操作都通过 `InvokeRequired` 检查
2. **性能影响**：`Refresh()` 调用会强制重绘，但影响很小
3. **异步方法**：`HideBottomProgress` 改为 async void，注意异常处理
4. **日志级别**：使用 DEBUG 级别，生产环境可以关闭

## 回滚方案

如果修复后出现问题，可以：
1. 移除 `MIN_PROGRESS_DISPLAY_TIME_MS` 相关代码
2. 移除 `Refresh()` 调用
3. 保留日志记录（有助于诊断）
