#### 3.2 添加拖拽事件处理

```csharp
// 拖拽相关变量
private Panel draggedRow = null;

// 鼠标按下事件（开始拖拽）
private void Row_MouseDown(object sender, MouseEventArgs e)
{
    if (e.Button == MouseButtons.Left && sender is Panel row)
    {
        draggedRow = row;
        SelectRow(row);
        row.DoDragDrop(row, DragDropEffects.Move);
    }
}

// 拖拽进入事件（显示拖拽效果）
private void Row_DragEnter(object sender, DragEventArgs e)
{
    if (e.Data.GetDataPresent(typeof(Panel)))
    {
        e.Effect = DragDropEffects.Move;
    }
}

// 拖拽放下事件（执行排序）
private void Row_DragDrop(object sender, DragEventArgs e)
{
    if (sender is Panel targetRow && draggedRow != null && draggedRow != targetRow)
    {
        // 获取拖拽行和目标行的索引
        int draggedIndex = pnlTextItems.Controls.IndexOf(draggedRow);
        int targetIndex = pnlTextItems.Controls.IndexOf(targetRow);

        if (draggedIndex != -1 && targetIndex != -1)
        {
            // 移除拖拽行
            pnlTextItems.Controls.Remove(draggedRow);

            // 在目标位置插入
            pnlTextItems.Controls.SetChildIndex(draggedRow, targetIndex);

            // 更新预览
            UpdateComboPreview();

            LogHelper.Debug($"拖拽排序: 从位置 {draggedIndex} 移动到 {targetIndex}");
        }

        draggedRow = null;
    }
}
```

## 技术细节

### WinForms 拖拽机制

**拖拽流程：**
1. **MouseDown** - 用户按下鼠标，开始拖拽
2. **DoDragDrop** - 启动拖拽操作
3. **DragEnter** - 鼠标进入目标控件
4. **DragDrop** - 用户释放鼠标，完成拖拽

**关键属性：**
- `AllowDrop = true` - 允许控件接受拖放
- `Cursor = Cursors.Hand` - 鼠标指针变为手型
- `DragDropEffects.Move` - 拖拽效果为移动

### 排序实现

**核心逻辑：**
```csharp
// 1. 移除拖拽行
pnlTextItems.Controls.Remove(draggedRow);

// 2. 在目标位置插入
pnlTextItems.Controls.SetChildIndex(draggedRow, targetIndex);

// 3. 更新预览
UpdateComboPreview();
```

**为什么使用 SetChildIndex？**
- `SetChildIndex` 可以精确控制控件在容器中的位置
- 比 `Insert` 方法更适合 FlowLayoutPanel
- 自动触发布局更新

### 视觉反馈

**选中状态：**
```csharp

```

**鼠标指针：**
```csharp

```

**拖拽效果：**

## 优势对比

### 上移下移按钮 vs 拖拽排序

| 特性 | 上移下移按钮 | 拖拽排序 | 优势 |
|------|-------------|---------|------|
| **操作步骤** | 多次点击 | 一次拖拽 | 拖拽 ✅ |
| **界面占用** | 需要按钮空间 | 无需额外空间 | 拖拽 ✅ |
| **用户体验** | 传统 | 现代化 | 拖拽 ✅ |
| **学习成本** | 低 | 中等 | 按钮 ✅ |
| **移动距离** | 受限（一次一步） | 任意位置 | 拖拽 ✅ |
| **视觉反馈** | 按钮状态 | 拖拽效果 | 拖拽 ✅ |

**结论：** 拖拽排序在大多数方面都优于上移下移按钮。
