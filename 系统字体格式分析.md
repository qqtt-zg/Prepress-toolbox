## 字体格式说明

### TTF (TrueType Font)
- **全称：** TrueType Font
- **特点：** 单一字体文件，包含一种字体样式
- **兼容性：** PDFsharp 支持良好 ✅
- **示例：** arial.ttf, simhei.ttf, calibri.ttf

### TTC (TrueType Collection)
- **全称：** TrueType Collection
- **特点：** 字体集合文件，包含多种字体样式（Regular、Bold、Italic 等）
- **兼容性：** PDFsharp 6.2.0 存在兼容性问题 ⚠️
- **示例：** msyh.ttc, simsun.ttc

### OTF (OpenType Font)
- **全称：** OpenType Font
- **特点：** 更现代的字体格式，支持更多特性
- **兼容性：** PDFsharp 支持 ✅
- **示例：** 部分现代字体

## TTC 字体兼容性问题

### 问题描述

**微软雅黑 (msyh.ttc)** 和 **宋体 (simsun.ttc)** 是 TTC 格式，在 PDFsharp 6.2.0 中存在兼容性问题：

```
错误类型: System.NullReferenceException
错误位置: PdfSharp.Fonts.OpenType.OpenTypeFontFace.CetOrCreateFrom(XFontSource fontSource)
原因: PDFsharp 在解析 TTC 文件时存在 bug
```

### 解决方案

我们已经实现了以下解决方案：

#### 1. 启用 Windows 字体支持
```csharp
GlobalFontSettings.UseWindowsFontsUnderWindows = true;
```
- 让 PDFsharp 使用 Windows 平台的字体解析器
- 平台解析器对 TTC 格式支持更好

#### 2. 自定义字体解析器作为回退
```csharp
GlobalFontSettings.FallbackFontResolver = new SystemFontResolver();
```
- 自定义解析器处理 TTF 和 OTF 格式
- TTC 格式返回 null，让平台解析器处理

#### 3. TTC 文件检测
```csharp
string extension = Path.GetExtension(fontFilePath).ToLower();
if (extension == ".ttc")
{
    // 返回 null，让 PDFsharp 使用平台解析器
    return null;
}
```

### 解析顺序

```
用户选择字体
    ↓
PDFsharp 尝试创建 XFont
    ↓
1. 平台解析器（处理 TTC）
    ↓
    成功 → 使用平台解析器
    ↓
    失败 ↓
2. 自定义解析器（处理 TTF/OTF）
    ↓
    成功 → 使用自定义解析器
    ↓
    失败 → 字体不兼容
```

## 技术细节

### TTC 文件结构

```
TTC 文件 (TrueType Collection)
├── 文件头 (TTC Header)
│   ├── 标识: "ttcf"
│   ├── 版本号
│   └── 字体数量
├── 字体 1 (Regular)
│   ├── 字体表
│   └── 字形数据
├── 字体 2 (Bold)
│   ├── 字体表
│   └── 字形数据
└── 字体 3 (Italic)
    ├── 字体表
    └── 字形数据
```

### TTF 文件结构

```
TTF 文件 (TrueType Font)
├── 文件头 (TTF Header)
│   ├── 标识: 0x00010000
│   └── 表数量
├── 字体表
│   ├── head (字体头)
│   ├── hhea (水平头)
│   ├── maxp (最大值)
│   ├── name (名称)
│   ├── cmap (字符映射)
│   └── glyf (字形数据)
└── 字形数据
```

### PDFsharp 解析问题

**TTC 文件的问题：**
1. PDFsharp 需要知道使用 TTC 中的哪个字体（索引）
2. 自定义解析器提供的字节数据缺少索引信息
3. PDFsharp 内部解析时出现 NullReferenceException

**解决方法：**
- 让平台解析器处理 TTC 文件
- 平台解析器知道如何正确解析 TTC 文件
- 自定义解析器只处理 TTF/OTF 文件
