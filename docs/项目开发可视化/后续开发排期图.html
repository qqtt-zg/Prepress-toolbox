后续开发排期图

按优先级排列的开发路线图 · P0 紧急 → P1 重要 → P2 锦上添花

P0 紧急 P1 重要 P2 可延后
P0 紧急

核心排版能力补全

建议：1-2 周
为什么优先：N-Up 是排版最基础能力，骑马订已有但 N-Up 缺失导致工作台面板功能不完整；Action List 是后续所有高级排版功能的架构基础，不先做这个后面全部要返工。
1

N-Up 多合一拼版

多页面拼合到单张，支持自定义行列数、间距、出血裁切

依赖：iText PDF 操作 · 影响：工作台面板完整性
2

Action List 动作列表系统

模块化 PDF 处理链，参考 PitStop Pro 设计，每个 Action 独立可组合

依赖：无 · 影响：所有后续排版功能的基础设施
P1 重要

高级排版与数据链路增强

建议：3-4 周
为什么排在 P1：这些功能依赖 P0 的 Action List 架构。Shuffle/Step&Repeat/Cut&Stack 都是独立排版模式，需要先有统一的 Action 基础设施才能高效开发。
3

裁切标记生成 (Cut Marks)

输出印刷裁切对位标记，支持自定义标记样式和位置

依赖：Action List 架构
4

页面重排 (Shuffle)

自定义页面顺序重新排列 PDF 页面，支持复杂重排规则

依赖：Action List 架构
5

Step & Repeat

单个图案重复排列，适用于标签、包装等场景

依赖：Action List + N-Up 基础
P2 可延后

专业化与架构优化

建议：5-8 周
为什么放最后：Cut & Stack 是小众场景；旧架构迁移是技术债务，不影响新功能使用，可在功能稳定后逐步推进。
6

Cut & Stack

按堆叠方式分切排列，适用于大批量印刷场景

依赖：Action List 架构
7

旧架构迁移 (Form1 → Panel)

将 Form1 单体窗体中的功能逐步迁移到 MainShellForm + Panel 架构

技术债务 · 不影响新功能 · 渐进式迁移
