## 使用方法

### 步骤 1: 打开字体设置

1. 启动应用程序
2. 进入设置界面
3. 找到"标识页字体"设置

### 步骤 2: 选择字体

1. 点击字体下拉列表
2. 浏览可用字体（中文字体优先显示）
3. 选择您想要的字体

### 步骤 3: 测试字体

1. 在预览框中输入测试文本
   - 建议输入实际的标识页内容
   - 例如：`威立德-氯氧铝-100F-黑白光膜-67x40Z`

2. 等待 1-2 秒，查看兼容性状态：
   - ✅ **绿色 "PDF兼容"** = 可以使用
   - ❌ **红色 "PDF不兼容"** = 需要更换

3. 如果显示绿色，说明字体完全可用

### 步骤 4: 保存设置

1. 确认字体显示为"✓ PDF兼容"
2. 点击保存按钮
3. 设置已生效

### 步骤 5: 生成 PDF

1. 使用正常流程生成 PDF
2. 标识页会使用您选择的字体
3. 文字显示效果与预览一致

## 常见问题

### Q1: 为什么有些字体还是不兼容？

**A:** 可能的原因：

1. **字体格式特殊**
   - 某些 PostScript 字体
   - 某些位图字体
   - 某些矢量字体

2. **字体文件问题**
   - 文件损坏
   - 格式错误
   - 缺少必要数据

3. **字体保护**
   - 有 DRM 保护
   - 需要特殊授权

**解决方法：**
- 选择常用字体（微软雅黑、宋体、Arial 等）
- 这些字体经过充分测试，兼容性最好

### Q2: 预览正常但提示不兼容？

**A:** 这是正常的。

- **WinForms 预览**：使用 Windows GDI 渲染
- **PDF 生成**：使用 PDFsharp 渲染
- 两者使用不同的渲染引擎

**建议：**
- 优先选择提示"✓ PDF兼容"的字体
- 这样可以确保预览效果与 PDF 效果一致

### Q3: 字体测试需要多长时间？

**A:** 通常 1-2 秒。

测试过程：
1. 创建 PDF 文档（~100ms）
2. 创建字体对象（~500ms）
3. 绘制和测量（~100ms）
4. 总计：~700ms - 1000ms

**注意：**
- 首次测试可能稍慢
- 后续测试会更快（有缓存）

### Q4: 可以使用自定义字体吗？

**A:** 可以，但需要满足条件：

1. **字体文件位置**
   - 必须安装到系统字体目录
   - 路径：`C:\Windows\Fonts`

2. **字体文件格式**
   - TrueType (.ttf)
   - TrueType Collection (.ttc)
   - OpenType (.otf)

3. **安装方法**
   - 右键字体文件
   - 选择"为所有用户安装"
   - 重启应用程序

### Q5: 字体列表太长，如何快速找到？

**A:** 字体列表已优化：

1. **中文字体优先**
   - 中文字体显示在列表顶部
   - 包括：微软雅黑、宋体、黑体等

2. **按字母排序**
   - 英文字体按字母顺序排列
   - 方便查找

3. **搜索功能**
   - 在下拉列表中输入字体名称
   - 自动过滤匹配项

## 技术细节

### 字体加载机制

1. **启动时扫描**
   - 扫描 `C:\Windows\Fonts` 目录
   - 支持 .ttf、.ttc、.otf 格式
   - 建立文件名到路径的映射

2. **延迟加载**
   - 只在需要时读取字体文件
   - 读取后缓存字节数据
   - 避免重复 I/O

3. **智能映射**
   - 显示名称 → 文件名
   - 支持中英文名称
   - 支持字体样式

5. 返回确定性结果

**测试代码与生产代码一致：**

## 故障排除

### 问题：字体列表为空

**可能原因：**
- 无法访问 `C:\Windows\Fonts` 目录
- 权限不足

**解决方法：**
1. 以管理员身份运行应用程序
2. 检查文件系统权限
3. 查看日志文件

### 问题：所有字体都不兼容

**可能原因：**
- 字体解析器初始化失败
- PDFsharp 库问题

**解决方法：**
1. 重启应用程序
2. 查看启动日志
3. 联系技术支持

### 问题：特定字体不兼容

**可能原因：**
- 字体文件损坏
- 字体格式不支持

**解决方法：**
1. 重新安装该字体
2. 选择其他字体
3. 使用推荐字体
