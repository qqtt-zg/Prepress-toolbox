# 修复记录 - 进度条位置调整

## 问题描述
进度条初始显示位置不在最左端，而是从 x=10 的位置开始。

## 原因分析
`statusPanel` 设置了 `Padding = new Padding(10, 0, 10, 0)`，导致使用 `Dock = DockStyle.Top` 的进度条受到 padding 影响，从 x=10 开始显示。

## 解决方案

### 1. 修改进度条布局方式
将 `Dock = DockStyle.Top` 改为 `Anchor = Top | Left | Right`，并设置绝对位置：

```csharp
// 修改前
this._progressOverlay.Dock = System.Windows.Forms.DockStyle.Top;
this._progressOverlay.Location = new System.Drawing.Point(10, 0);
this._progressOverlay.Size = new System.Drawing.Size(920, 4);

// 修改后
this._progressOverlay.Anchor = AnchorStyles.Top | AnchorStyles.Left | AnchorStyles.Right;
this._progressOverlay.Location = new System.Drawing.Point(0, 0);
this._progressOverlay.Size = new System.Drawing.Size(940, 4);
```

### 2. 动态调整宽度
在构造函数中添加事件处理，确保窗口大小改变时进度条宽度自动调整：

```csharp
public PdfOperationsPanel()
{
    InitializeComponent();
    
    // 确保进度条在最上层
    _progressOverlay.BringToFront();
    
    // 监听状态栏大小变化，动态调整进度条宽度
    statusPanel.SizeChanged += (s, e) =>
    {
        _progressOverlay.Width = statusPanel.Width;
    };
}
```

## 技术要点

### Dock vs Anchor
- **Dock**: 控件会受到父容器 Padding 的影响
- **Anchor**: 控件使用绝对位置，不受 Padding 影响

### 为什么使用 Anchor 而不是移除 Padding
保持 `statusPanel` 的 Padding 是为了让状态文字 `_lblStatus` 有合适的左边距，不紧贴边缘。只有进度条需要从最左端开始。

### BringToFront 的作用
确保进度条在所有控件的最上层（Z-order），这样它才能正确地叠加在状态栏上方。
