# 修复事件分组内节点拖拽功能

## 问题描述
事件分组的分组内节点位置拖拽功能失效，用户无法在同一分组内对项目进行排序。

## 问题原因分析
在之前的代码中，拖拽功能只实现了：
1. **分组之间的排序** - 可以拖拽分组节点改变分组顺序
2. **跨分组的项目移动** - 可以将项目从一个分组拖拽到另一个分组

但是缺少了：
3. **分组内项目排序** - 在同一分组内拖拽项目改变项目顺序

## 修复内容

### 文件修改
**文件路径**: `src/WindowsFormsApp3/Controls/EventGroupsTreeView.cs`

## 技术实现细节

### 拖拽检测流程
```
用户开始拖拽项目节点
    ↓
DragEnter 检测拖拽类型
    ↓
判断目标节点类型
    ├─ 分组节点 → 跨分组移动
    ├─ 同分组项目节点 → 分组内排序
    └─ 其他 → 默认处理
    ↓
设置相应的 DragDropEffects
```

### 分组内排序算法
```
获取分组内所有项目节点
    ↓
计算拖拽节点和目标节点的索引
    ↓
基于鼠标位置计算插入位置
    ↓
执行节点移动操作
    ↓
触发排序完成事件
```

### 插入位置计算
- **向上拖拽**: 根据鼠标位置相对于目标节点中点的位置决定插入位置
- **向下拖拽**: 同样基于鼠标位置进行精确计算
- **边界处理**: 确保插入索引在有效范围内
