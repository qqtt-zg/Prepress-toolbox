# 字体检测对比测试指南

## 测试目的

对比 iText 和 Poppler 两种字体检测方案的准确性，验证 Poppler 方案是否与 Adobe Acrobat 结果一致。

## 测试准备

### 1. 安装 Poppler 工具

**下载地址：**
https://github.com/oschwartz10612/poppler-windows/releases

**安装步骤：**
```
1. 下载 Release-24.08.0-0.zip (约 20MB)
2. 解压到项目根目录
3. 确保路径为: poppler\bin\pdffonts.exe
```

**验证安装：**
```cmd
cd scripts
test_pdffonts.bat
```

### 2. 准备测试文件

准备包含以下特征的 PDF 文件：
- ✅ 嵌入字体（完全嵌入）
- ✅ 子集字体（部分嵌入）
- ✅ 未嵌入字体
- ✅ 中文字体（CID 字体）
- ✅ 英文字体（Type 1, TrueType）
- ✅ 特殊字体（Type 3）

## 测试步骤

### 步骤 1: Adobe Acrobat 基准测试

1. 用 Adobe Acrobat DC 打开测试 PDF
2. 点击 **文件 → 属性 → 字体**
3. 记录所有字体信息：
   - 字体名称
   - 字体类型
   - 编码方式
   - 嵌入状态（嵌入/嵌入子集/未嵌入）

**记录格式：**
```
文件名: test.pdf
总字体数: 5

1. SimSun (嵌入子集)
   类型: Type 0 (CID)
   编码: Identity-H

2. Arial-Bold (嵌入子集)
   类型: TrueType
   编码: WinAnsi

3. Times-Roman (未嵌入)
   类型: Type 1
   编码: WinAnsi
```

### 步骤 2: Poppler 命令行测试

```cmd
cd poppler\bin
pdffonts.exe "..\..\官方示例文件\test.pdf"
```

**记录输出：**
```
name                                 type              encoding         emb sub uni object ID
------------------------------------ ----------------- ---------------- --- --- --- ---------
ABCDEE+SimSun                        CID Type 0C       Identity-H       yes yes yes      8  0
BCDEFG+Arial-Bold                    TrueType          WinAnsi          yes yes yes     12  0
Times-Roman                          Type 1            WinAnsi          no  no  no      15  0
```

### 步骤 3: iText 检测测试

1. 运行当前程序
2. 打开测试 PDF
3. 点击"检查器"按钮
4. 切换到"字体"标签页
5. 记录检测结果

### 步骤 4: Poppler 集成测试

1. 修改代码使用 Poppler 服务
2. 重新编译运行
3. 打开测试 PDF
4. 点击"检查器"按钮
5. 切换到"字体"标签页
6. 记录检测结果

## 附录：常见字体类型

### PostScript 字体
- **Type 1**: 标准 PostScript 字体
- **Type 1C**: 压缩的 Type 1 (CFF)
- **Type 3**: 用户定义字体（位图）

### TrueType 字体
- **TrueType**: 标准 TrueType 字体
- **TrueType (CID)**: CID-keyed TrueType

### CID 字体（中日韩）
- **CID Type 0**: CID PostScript 字体
- **CID Type 0C**: CID PostScript CFF 字体
- **CID TrueType**: CID TrueType 字体

### OpenType 字体
- **OpenType (CFF)**: 基于 CFF 的 OpenType
- **OpenType (TrueType)**: 基于 TrueType 的 OpenType
