# 字体预览主题响应问题修复记录

## 问题描述

在标识页字体设置界面中，`txtFontPreview` 字体预览输入框的文字颜色在程序重启后无法正确响应主题模式：
- 首次切换主题时，文字颜色能正确响应（深色主题显示浅色文字，浅色主题显示黑色文字）
- 但是设置主题后重启程序，`txtFontPreview` 的文字颜色不再响应主题，始终显示为黑色

## 问题原因

### 根本原因
主题应用的时序问题导致自定义主题颜色被覆盖：

1. **控件初始化阶段**：
   - `SettingsFontTextControl` 构造函数执行
   - `Load` 事件触发，调用 `ApplyThemeToControls()` 设置自定义颜色

2. **父窗体主题应用阶段**（在 Load 之后）：
   - `MainShellForm` 在启动时调用 `ThemeHelper.ApplyTheme(this, isDark)`
   - `ThemeHelper` 递归遍历所有子控件，包括 `txtFontPreview`
   - 因为 `txtFontPreview` 是 `AntdUI.Input` 类型，`ThemeHelper.ApplyToAntdUIControl()` 方法会设置其 `ForeColor` 为 `_currentTheme.TextPrimary`
   - 这个操作**覆盖**了之前在 `Load` 事件中设置的自定义颜色

### 代码证据

在 `ThemeHelper.cs` 中：
```csharp
// AntdUI.Input
if (typeName == "Input")
{
    SetPropertySafe(control, "BackColor", _currentTheme.SurfaceLight);
    SetPropertySafe(control, "ForeColor", _currentTheme.TextPrimary);  // ← 这里覆盖了自定义颜色
    SetPropertySafe(control, "BorderColor", _currentTheme.Border);
    SetPropertySafe(control, "PlaceholderColor", _currentTheme.TextSecondary);
}
```

## 解决方案

### 方案概述
在 `ThemeHelper` 应用主题**之后**，重新应用自定义主题颜色。

### 实现方法
使用 `VisibleChanged` 事件，在控件变为可见时（此时 `ThemeHelper` 已完成主题应用）重新应用自定义颜色：

```csharp
public SettingsFontTextControl()
{
    InitializeComponent();

    if (LicenseManager.UsageMode != LicenseUsageMode.Designtime)
    {
        LoadSettings();
        
        // 在控件加载完成后刷新主题
        this.Load += (s, e) =>
        {
            RefreshRowThemes();
            ApplyThemeToControls();
        };

        // 在控件可见时重新应用主题（确保在 ThemeHelper 应用后执行）
        this.VisibleChanged += (s, e) =>
        {
            if (this.Visible)
            {
                // 延迟应用，确保 ThemeHelper 已完成
                System.Threading.Tasks.Task.Delay(50).ContinueWith(_ =>
                {
                    if (this.InvokeRequired)
                    {
                        this.Invoke(new Action(() =>
                        {
                            ApplyThemeToControls();
                            RefreshRowThemes();
                        }));
                    }
                    else
                    {
                        ApplyThemeToControls();
                        RefreshRowThemes();
                    }
                });
            }
        };
    }
}
```

### 改进 `ApplyThemeToControls` 方法
添加了更多的安全检查和强制刷新：

```csharp
private void ApplyThemeToControls()
{
    try
    {
        Color textColor = GetTextColor();
        
        // 应用主题到字体预览输入框
        if (txtFontPreview != null && !txtFontPreview.IsDisposed)
        {
            txtFontPreview.ForeColor = textColor;
            // 强制刷新显示
            txtFontPreview.Invalidate();
        }

        // 应用主题到组合预览输入框
        if (txtComboPreview != null && !txtComboPreview.IsDisposed)
        {
            txtComboPreview.ForeColor = textColor;
            // 强制刷新显示
            txtComboPreview.Invalidate();
        }

        LogHelper.Debug($"应用主题到控件，文字颜色: {textColor}, 是否深色主题: {IsDarkTheme()}");
    }
    catch (Exception ex)
    {
        LogHelper.Error($"应用主题到控件失败: {ex.Message}", ex);
    }
}
```

## 技术要点

### 1. 事件时序
- `Load` 事件：控件加载时触发，但此时父窗体可能还未应用主题
- `VisibleChanged` 事件：控件可见性改变时触发，在父窗体主题应用之后
- 使用 50ms 延迟确保 `ThemeHelper` 完成递归应用

### 2. 线程安全
- 使用 `InvokeRequired` 检查是否需要跨线程调用
- 使用 `Invoke` 确保 UI 更新在主线程执行

### 3. 控件状态检查
- 检查 `!IsDisposed` 避免访问已释放的控件
- 使用 `Invalidate()` 强制控件重绘

### 4. 主题判断逻辑
```csharp
private bool IsDarkTheme()
{
    if (this.ParentForm != null)
    {
        var bgColor = this.ParentForm.BackColor;
        // 计算亮度：(R * 299 + G * 587 + B * 114) / 1000
        int brightness = (bgColor.R * 299 + bgColor.G * 587 + bgColor.B * 114) / 1000;
        return brightness < 128;  // 亮度小于128认为是深色主题
    }
    return false;
}
```

1. 自定义主题颜色在 `ThemeHelper` 应用后仍然生效
2. 程序重启后主题响应正确
3. 主题切换时实时响应
4. 线程安全和控件状态安全
