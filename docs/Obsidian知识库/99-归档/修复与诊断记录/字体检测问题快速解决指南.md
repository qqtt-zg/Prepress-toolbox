# 字体检测问题快速解决指南

## 🎯 核心问题

**症状**：Poppler检测的字体嵌入状态与Adobe Acrobat不一致

**根本原因**：pdffonts输出使用固定列宽格式，而不是简单的空格分隔

**解决方案**：使用固定位置解析代替正则表达式分割

---

## ⚡ 快速修复（5分钟）

### 步骤1：替换解析方法

打开 `src/WindowsFormsApp3/Services/PdfFontInspectorService_Poppler.cs`

找到 `ParseFontLine` 方法，替换为：

```csharp
private FontInfo ParseFontLine(string line)
{
    if (line.Length < 75) return null;

    // 按固定列宽提取
    string name = line.Substring(0, 37).Trim();
    string type = line.Substring(37, 18).Trim();
    string encoding = line.Substring(55, 17).Trim();
    string emb = line.Substring(72, 4).Trim();
    string sub = line.Substring(76, 4).Trim();

    var fontInfo = new FontInfo
    {
        FontName = name,
        FontSubtype = type,
        Encoding = encoding
    };

    // 判断嵌入状态
    if (emb.Equals("yes", StringComparison.OrdinalIgnoreCase))
    {
        fontInfo.EmbeddingStatus = sub.Equals("yes", StringComparison.OrdinalIgnoreCase)
            ? FontEmbeddingStatus.SubsetEmbedded
            : FontEmbeddingStatus.FullyEmbedded;
        fontInfo.IsSubset = sub.Equals("yes", StringComparison.OrdinalIgnoreCase);
    }
    else
    {
        fontInfo.EmbeddingStatus = FontEmbeddingStatus.NotEmbedded;
    }

    // 移除子集前缀
    if (Regex.IsMatch(fontInfo.FontName, @"^[A-Z]{6}\+"))
    {
        fontInfo.FontName = fontInfo.FontName.Substring(7);
    }

    fontInfo.IsStandardFont = IsStandardFont(fontInfo.FontName);
    DetectFontIssues(fontInfo);

    return fontInfo;
}
```

## 📊 常见差异模式

### 模式1：子集嵌入误判为未嵌入

**pdffonts输出**：
```
EPROCH+MicrosoftYaHei    CID Type 0C    Identity-H    yes yes yes
```

**错误解析**：
- 嵌入状态：NotEmbedded ❌

**正确解析**：
- 嵌入状态：SubsetEmbedded ✓
- 原因：emb=yes, sub=yes

### 模式2：字体名称包含子集前缀

**pdffonts输出**：
```
ABCDEF+Arial-Bold
```

**错误显示**：
- 字体名称：ABCDEF+Arial-Bold ❌

**正确显示**：
- 字体名称：Arial-Bold ✓
- 原因：需要移除 `^[A-Z]{6}\+` 前缀

### 模式3：字体类型名称差异

| Poppler | Acrobat | 说明 |
|---------|---------|------|
| CID Type 0C | CIDFont Type 0 (CFF) | 需要映射 |
| Type 1C | Type 1 (CFF) | 需要映射 |
| TrueType | TrueType | 一致 |

## 🛠️ 修正规则速查

### 规则1：列位置解析

```
位置:  0         37        55        72  76  80
       |         |         |         |   |   |
       name──────┘type─────┘encoding─┘emb┘sub┘uni
```

### 规则2：嵌入状态判断

```
emb=yes + sub=yes → SubsetEmbedded (子集嵌入)
emb=yes + sub=no  → FullyEmbedded (完全嵌入)
emb=no  + sub=no  → NotEmbedded (未嵌入)
```

### 规则3：子集前缀移除

```
正则: ^[A-Z]{6}\+
示例: EPROCH+MicrosoftYaHei → MicrosoftYaHei
```

### 规则4：标准字体识别

```
标准字体（未嵌入也不报错）:
- Times-Roman, Times-Bold, Times-Italic, Times-BoldItalic
- Helvetica, Helvetica-Bold, Helvetica-Oblique, Helvetica-BoldOblique
- Courier, Courier-Bold, Courier-Oblique, Courier-BoldOblique
- Symbol, ZapfDingbats
```

## 🐛 故障排查

### 问题1：字体数量为0

**症状**：
```
[Poppler] 解析完成，共 0 个字体
```

**检查**：
1. PDF是否已转曲？
2. pdffonts是否有输出？
3. 行长度是否足够（≥75字符）？

**解决**：
```cmd
# 手动测试pdffonts
poppler\bin\pdffonts.exe "测试文件.pdf"
```

### 问题2：嵌入状态仍然错误

**症状**：
```
emb='yes', sub='yes' 但显示 NotEmbedded
```

**检查**：
1. 是否使用了新的解析方法？
2. 列位置是否正确（72, 76）？
3. 字符串比较是否忽略大小写？

**解决**：
```csharp
// 确保使用 OrdinalIgnoreCase
emb.Equals("yes", StringComparison.OrdinalIgnoreCase)
```

### 问题3：pdffonts未找到

**症状**：
```
[Poppler] ✗ 未找到 pdffonts 工具
```

**检查**：
1. poppler文件夹是否存在？
2. 是否复制到编译输出目录？

**解决**：
```cmd
# 复制到编译目录
xcopy /E /I /Y poppler src\WindowsFormsApp3\bin\Debug\net48\win-x64\poppler
```
