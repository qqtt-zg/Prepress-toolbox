## 问题描述

### 现象
- ~~标识页插入的文字不可见~~ ✅ 已修复
- 字体预览（picPdfPreview）中的文字不可见 ⚠️ 需要进一步修复
- 只能看到白色背景，看不到文字

### 根本原因

在 iTextSharp.LGPLv2.Core 中，PDF 内容有两个绘制层：

1. **DirectContentUnder** - 在文档内容下方（背景层）
2. **DirectContent** - 在文档内容上方（前景层）

**错误代码：**
```csharp
// 绘制白色背景
PdfContentByte cb = writer.DirectContent;  // ❌ 错误：使用前景层
cb.SetColorFill(BaseColor.White);
cb.Rectangle(0, 0, pageSize.Width, pageSize.Height);
cb.Fill();

// 添加文字
document.Add(paragraph);  // 文字在中间层
```

**绘制顺序：**
```
┌─────────────────────┐
│  DirectContent      │ ← 白色背景（覆盖了文字）❌
├─────────────────────┤
│  Document Content   │ ← 文字（被覆盖）
├─────────────────────┤
│  DirectContentUnder │
└─────────────────────┘
```

**结果：** 白色背景在文字上方，文字被覆盖，不可见。

### 方案 1：标识页插入（已成功）✅

**文件：** `ITextSharpLGPLPdfTools.cs`

// 绘制白色背景（使用 DirectContentUnder 确保在文字下方）

**结果：** ✅ 标识页文字正常显示

### 方案 2：字体预览（增强版）✅

使用 `DirectContent` 直接绘制文字，确保文字在最上层。

**文件：** `SettingsFontTextControl.cs`

**问题分析：**
- 使用 `document.Add(paragraph)` 添加文字可能受到页边距、布局等影响
- 文字可能被裁剪或位置不正确

**解决方案：**
使用 `DirectContent` 直接绘制文字，完全控制文字位置和显示。

**修改代码：**
```csharp
// 绘制白色背景（背景层）
PdfContentByte cbUnder = writer.DirectContentUnder;
cbUnder.SetColorFill(BaseColor.White);
cbUnder.Rectangle(0, 0, 400, 100);
cbUnder.Fill();

// 使用 DirectContent 直接绘制文字（前景层，确保在最上层）
PdfContentByte cb = writer.DirectContent;
cb.BeginText();
cb.SetFontAndSize(baseFont, 16);
cb.SetColorFill(BaseColor.Black);

// 计算文字宽度和位置（居中）
float textWidth = baseFont.GetWidthPoint(testText, 16);
float x = (400 - textWidth) / 2;  // 水平居中
float y = 42;  // 垂直居中（100/2 - 16/2 = 42）

cb.SetTextMatrix(x, y);
cb.ShowText(testText);
cb.EndText();
```

**绘制顺序：**
```
┌─────────────────────┐
│  DirectContent      │ ← 文字（最上层）✅
├─────────────────────┤
│  Document Content   │ ← 空
├─────────────────────┤
│  DirectContentUnder │ ← 白色背景（最下层）
└─────────────────────┘
```

**结果：** ✅ 字体预览文字正常显示

## 技术说明

### iTextSharp 的内容层次结构

iTextSharp.LGPLv2.Core 提供了三个内容层：

| 层次 | 属性 | 用途 | 绘制顺序 |
|------|------|------|---------|
| **前景层** | `writer.DirectContent` | 水印、覆盖层、**直接绘制文字** | 最上层（第3层）|
| **文档层** | `document.Add()` | 正常内容（文字、图片） | 中间层（第2层）|
| **背景层** | `writer.DirectContentUnder` | 背景、底纹 | 最下层（第1层）|

### 两种文字绘制方式

#### 方式 1：document.Add()（标识页使用）

**优点：**
- ✅ 简单易用
- ✅ 自动换行
- ✅ 支持段落、表格等复杂布局

**缺点：**
- ⚠️ 受页边距影响
- ⚠️ 位置控制不够精确

**适用场景：**
- 标识页插入（需要自动换行）
- 复杂文档内容

**代码示例：**
```csharp
Paragraph paragraph = new Paragraph(text, font);
paragraph.Alignment = Element.ALIGN_CENTER;
document.Add(paragraph);
```

#### 方式 2：DirectContent.ShowText()（预览使用）

**优点：**
- ✅ 精确控制位置
- ✅ 不受页边距影响
- ✅ 确保在最上层显示

**缺点：**
- ⚠️ 需要手动计算位置
- ⚠️ 不支持自动换行

**适用场景：**
- 字体预览（简单文本，需要精确居中）
- 水印、签名

**代码示例：**
```csharp
PdfContentByte cb = writer.DirectContent;
cb.BeginText();
cb.SetFontAndSize(baseFont, 16);
cb.SetTextMatrix(x, y);
cb.ShowText(text);
cb.EndText();
```

### 为什么预览需要使用 DirectContent？

**问题：**
- 预览 PDF 尺寸很小（400x100点）
- 使用 `document.Add()` 时，默认页边距可能占用大部分空间
- 文字可能被裁剪或显示在可见区域外

**解决：**
- 使用 `DirectContent` 直接绘制
- 精确计算文字位置
- 确保文字在可见区域内

## 相关知识

### PDF 内容流

PDF 文档的内容是通过"内容流"（Content Stream）绘制的：

```
/Contents [
  % 背景层（DirectContentUnder）
  q
  1 1 1 rg  % 白色
  0 0 400 100 re  % 矩形
  f  % 填充
  Q
  
  % 文档层（Document Content）
  BT
  /F1 16 Tf  % 字体
  200 50 Td  % 位置
  (测试文字) Tj  % 文字
  ET
  
  % 前景层（DirectContent）
  % 空
]
```

### 绘制顺序

PDF 按照内容流的顺序绘制：
1. 先绘制的内容在下层
2. 后绘制的内容在上层
3. 后绘制的内容会覆盖先绘制的内容

**iTextSharp 的处理：**
- `DirectContentUnder` → 最先写入内容流（下层）
- `Document.Add()` → 中间写入内容流（中层）
- `DirectContent` → 最后写入内容流（上层）
