## 功能概览

### 核心功能
1. **字体转曲**: 将PDF中的文字转换为路径（曲线）
2. **预览效果**: 转曲后自动预览效果
3. **保存**: 覆盖原文件
4. **另存为**: 保存到新文件
5. **撤回**: 恢复到原始PDF（新增！）

### 工具栏布局
```
[打开PDF] [保存] [另存为] [撤回] | 印刷标记: [添加裁切标记] [添加套准标记] | [检查器]
```

## 完整工作流

### 标准流程
```
1. 打开PDF文件
   ↓
2. 点击"检查器"按钮
   ↓
3. 在检查器的"字体"标签页点击"转曲"按钮
   ↓
4. 确认转曲操作
   ↓
5. 等待转曲完成（自动预览）
   ↓
6. 选择操作：
   - 保存：覆盖原文件
   - 另存为：保存到新文件
   - 撤回：恢复原始PDF
```

### 使用场景

#### 场景1: 标准转曲并保存
```
打开PDF → 转曲 → 预览满意 → 保存
```

#### 场景2: 转曲并保留原文件
```
打开PDF → 转曲 → 预览满意 → 另存为
```

#### 场景3: 转曲后撤回
```
打开PDF → 转曲 → 预览不满意 → 撤回 → 重新调整 → 再次转曲
```

#### 场景4: 对比效果
```
打开PDF → 转曲 → 查看转曲效果 → 撤回 → 查看原始效果 → 决定是否转曲
```

## 按钮功能详解

### 1. 转曲按钮（检查器中）
- **位置**: 检查器窗口 → 字体标签页 → 工具栏
- **功能**: 将PDF中的文字转换为路径
- **启用条件**: 加载PDF后
- **操作结果**: 
  - 在内存中生成转曲后的PDF
  - 自动预览转曲效果
  - 启用"保存"、"另存为"、"撤回"按钮
  - 状态栏显示"已转曲 (未保存)"

### 2. 保存按钮（工具栏）
- **位置**: 主窗口工具栏
- **功能**: 覆盖原文件
- **启用条件**: 有未保存的转曲更改
- **操作结果**:
  - 将转曲后的PDF写入原文件
  - 重新加载文件
  - 禁用"保存"和"撤回"按钮
  - 状态栏显示"已保存"

### 3. 另存为按钮（工具栏）
- **位置**: 主窗口工具栏
- **功能**: 保存到新文件
- **启用条件**: 加载PDF后（始终可用）
- **默认文件名**: 
  - 有转曲更改: `原文件名_转曲.pdf`
  - 无转曲更改: `原文件名_副本.pdf`
- **操作结果**:
  - 弹出保存对话框
  - 保存到指定位置
  - 询问是否打开新文件

### 4. 撤回按钮（工具栏）
- **位置**: 主窗口工具栏
- **功能**: 撤回转曲操作，恢复原始PDF
- **启用条件**: 有未保存的转曲更改
- **操作结果**:
  - 弹出确认对话框
  - 清除转曲数据
  - 重新加载原文件
  - 禁用"保存"和"撤回"按钮
  - 状态栏显示"已加载"

## 技术架构

### 数据流
```
原始PDF文件
    ↓
Ghostscript转曲
    ↓
转曲后的PDF字节数组（内存）
    ↓
临时文件（预览用）
    ↓
用户选择：
  - 保存 → 写入原文件
  - 另存为 → 写入新文件
  - 撤回 → 丢弃数据，重新加载原文件
```

### 事件流
```
PdfInspectorControl (转曲按钮)
    ↓ FontOutlineCompleted 事件
PdfInspectorForm (转发事件)
    ↓ FontOutlineCompleted 事件
PdfOperationsPanel (处理事件)
    ↓ 保存字节数组、启用按钮、更新预览
用户操作：
    ↓ 保存/另存为/撤回
PdfOperationsPanel (执行操作)
```

### 状态管理
```csharp
// 核心状态变量
private string _currentFilePath;           // 当前文件路径
private byte[] _modifiedPdfBytes;          // 转曲后的PDF字节数组
private bool _hasUnsavedChanges;           // 是否有未保存的更改

// 状态转换
加载文件: _currentFilePath = path, _modifiedPdfBytes = null, _hasUnsavedChanges = false
转曲完成: _modifiedPdfBytes = bytes, _hasUnsavedChanges = true
保存: _modifiedPdfBytes = null, _hasUnsavedChanges = false
撤回: _modifiedPdfBytes = null, _hasUnsavedChanges = false
```

## 实现细节

### 转曲实现
- **引擎**: Ghostscript 10.06.0
- **参数**: `-dNoOutputFonts -sDEVICE=pdfwrite -dPDFSETTINGS=/prepress`
- **方法**: `PdfFontOutlineService.ConvertTextToOutlinesBytes()`
- **返回**: `byte[]` 转曲后的PDF字节数组

### 预览实现
- **方式**: 创建临时文件
- **路径**: `%TEMP%\temp_preview_{GUID}.pdf`
- **加载**: 使用CefSharp加载临时文件
- **清理**: 加载新文件时自动清理

### 撤回实现
- **方式**: 清除内存数据，重新加载原文件
- **确认**: 弹出确认对话框防止误操作
- **恢复**: 完全恢复到转曲前的状态

## 安全机制

### 1. 数据保护
- 转曲操作不修改原文件
- 只在用户明确保存时才写入文件
- 撤回操作不会丢失原文件

### 2. 确认对话框
- 转曲前确认："字体转曲后将无法编辑文字内容，是否继续？"
- 撤回前确认："确定要撤回转曲操作吗？转曲后的内容将丢失。"
- 保存前无需确认（用户主动操作）

### 3. 状态一致性
- 所有操作后状态正确更新
- 按钮状态与实际状态一致
- 状态栏显示准确信息
