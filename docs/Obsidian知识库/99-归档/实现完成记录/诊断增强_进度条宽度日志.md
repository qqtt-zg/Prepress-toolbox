## 关键诊断点

### 1. statusPanel 宽度
- 如果 statusPanel 宽度为 0 或很小，说明布局有问题
- 正常情况下应该是窗口宽度（例如 940px）

### 2. 进度条位置
- 应该是 10（左侧 padding）
- 如果不是 10，说明 Dock 设置有问题

### 3. 计算的宽度
- 10% 应该约为 (statusPanel宽度 - 20) * 0.1
- 100% 应该约为 (statusPanel宽度 - 20)
- 如果宽度为 0，说明计算有问题

### 4. 可见性
- 应该显示 `可见: True`
- 如果是 False，说明 Visible 属性没有正确设置

## 可能的问题

### 问题 1：statusPanel 宽度为 0
**原因**：面板还没有完成布局
**解决**：在显示进度条前调用 `statusPanel.PerformLayout()`

### 问题 2：进度条宽度计算为 0
**原因**：statusPanel.Width 太小或为 0
**解决**：使用固定宽度或等待布局完成

### 问题 3：进度条位置不对
**原因**：Dock 设置冲突
**解决**：改用 Anchor 或手动设置位置

### 问题 4：Z-order 问题
**原因**：进度条在其他控件后面
**解决**：调用 `_progressBar.BringToFront()`
