# Obsidian知识库

这是为 `Prepress-toolbox` 项目整理的 Obsidian 版知识库骨架。

## 目标
- 为全项目提供统一知识入口
- 建立文档到代码的可追踪映射
- 区分“当前有效”“历史记录”“待整理”
- 用双链收口重复概念和冲突定义

## 推荐使用方式
1. 先打开 [Obsidian知识库图谱](../项目开发可视化/Obsidian知识库图谱.html)，快速理解主线、模块关系和代码入口
2. 从 [[项目总览]] 开始
3. 再看 [[模块地图]] 和 [[术语表]]
4. 进入具体模块页，例如 [[工作台与排版工作区]]、[[PDF检查器]]
5. 需要核对代码时，查看 [[代码入口与模块映射]]

## 目录说明
- `00-首页与导航`：总入口、MOC、术语、规则
- `01-产品与范围`：产品定位、范围、主线问题
- `02-模块知识库`：按模块组织的主知识页
- `03-架构与实现`：代码映射、架构约束、实现入口
- `04-调研与竞品`：市场调研、竞品复刻、需求来源
- `05-操作手册与流程`：用户手册、操作流程
- `06-发布与变更`：版本、发布、修复记录
- `07-测试与验证`：测试指南、验证结论
- `99-归档`：过期文档、历史阶段总结

## 当前状态
这是一层新的知识组织结构，先建立入口和映射，不直接搬动旧文档。后续迁移以“链接旧文档 -> 统一模块页 -> 再归档”为顺序。
