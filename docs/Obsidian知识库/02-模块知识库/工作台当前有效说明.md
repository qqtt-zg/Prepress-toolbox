# 工作台当前有效说明

## 当前有效定义
在当前项目语境下，“工作台”不应再被简单理解为单一面板，而应理解为一条包含两层语义的工作流主线：

1. **工作台壳层**
   - 对应当前导航入口的作业视图
   - 强调多 PDF 文件管理、预览、日志、作业队列
   - 代码上最接近 `AeWorkspacePanel`

2. **排版工作区**
   - 对应专业拼版与参数配置空间
   - 强调骑马订、N-Up、页面重排、裁切标记、生成 PDF
   - 代码上最接近 `ImpositionWorkspacePanel`

## 为什么这样定义
这是当前唯一能同时解释“代码现状”和“文档意图”的方式：
- 用户手册描述了 AE 风格工作台
- PRD、开发文档和调研报告描述的是专业排版工作区
- 两者都真实存在，但表达层级不同

因此，知识库中将其视为：
- **工作台** = 用户入口与作业壳层
- **排版工作区** = 工作台中的专业拼版子空间

## 当前已实现能力
### 已明确可用
- 骑马订拼版
- PDF 文件加载与多文件列表
- PDF 预览
- 基本作业切换

### 已有骨架但未完善
- N-Up 参数面板与生成流程
- 页面重排参数面板
- 裁切标记集成
- 进度条与执行状态

### 仍属规划项
- Cut & Stack
- Step & Repeat 增强
- 混拼
- DXF 刀线导出
- 热文件夹自动化
- Excel 订单驱动拼版

## 当前结论
后续文档和设计都应避免继续把“工作台”只写成一个单一 UI 面板名。更准确的写法是：
- 对用户入口讨论时，使用“工作台”
- 对专业拼版功能讨论时，使用“排版工作区”
- 对代码实现讨论时，明确写 `AeWorkspacePanel` 或 `ImpositionWorkspacePanel`
