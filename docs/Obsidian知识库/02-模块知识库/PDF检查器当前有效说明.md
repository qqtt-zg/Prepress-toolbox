# PDF检查器当前有效说明

## 当前有效定义
`PDF检查器` 当前应被理解为一个"检查优先"的 PDF 诊断入口，用于加载文档、查看页面与字体信息，并在需要时触发字体转曲等处理动作。

它包含两层职责：

1. **PDF 检查入口**
   - 加载 PDF、展示基础信息与检查结果
   - 重点是页面尺寸、页面边界框（MediaBox / CropBox / TrimBox / BleedBox / ArtBox）、字体信息、文档状态等可视化检查

2. **后处理承载入口**
   - 某些处理动作可以从这里发起，例如字体转曲
   - 处理结果不再等同于"立刻另存为完成文件"
   - 当前更接近"在当前工作上下文中生成结果，再由用户决定保存"

## 当前有效工作流
1. 打开 PDF
2. 查看检查信息（页面、字体、文档属性）
3. 需要时执行字体转曲
4. 转曲结果回到当前工作上下文
5. 用户再执行"保存"或"另存为"

## 当前已明确可用
- PDF 文件加载
- 文档基础信息查看
- 页面相关信息检查（页面边界框）
- 字体信息检查
- 从检查流程进入字体转曲处理
- 处理完成后保留结果供用户后续保存

## 当前需要覆盖旧认知的关键点

以下旧认知不应再作为"当前有效行为"理解：
- "转曲完成后立即弹出保存对话框"
- "字体转曲本身等于最终导出完成"

更准确的当前写法应是：
- **转曲完成 = 获得新的 PDF 结果**
- **文件落盘 = 用户后续主动保存**

## 与其他模块的边界
- 与 [[字体兼容与转曲]] 的关系：
  - `PDF检查器` 是入口与承载界面
  - `字体兼容与转曲` 是具体问题域与处理能力
- 与 [[工作台与排版工作区]] 的关系：
  - 都属于 PDF 工作流的一部分
  - 但前者偏检查与纠错，后者偏拼版与输出
