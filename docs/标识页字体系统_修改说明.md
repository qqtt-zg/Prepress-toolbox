# 标识页字体系统修改说明

## 修改日期
2026-01-12

## 问题描述
1. iText7 对 TrueType Collection (.ttc) 字体文件支持不完善，导致中文字体（如微软雅黑、宋体）显示为乱码
2. 原有字体选择限制在5种固定字体，无法使用系统其他字体
3. 字体预览与实际 PDF 生成效果不一致

## 解决方案

### 1. 字体选择界面改进 (SettingsFontTextControl.cs)

**修改内容：**
- 使用 `InstalledFontCollection` 加载系统所有已安装字体
- **在加载时验证字体与 PDFsharp 的兼容性**，只显示兼容的字体
- 中文/CJK 字体优先显示在列表顶部
- 字体预览同时测试 WinForms 和 PDFsharp 渲染效果
- 不兼容的字体会显示红色警告

**关键代码：**
```csharp
// 加载字体时验证兼容性
foreach (FontFamily font in sortedFonts)
{
    if (IsFontCompatibleWithPdfSharp(font.Name))
    {
        _systemFonts.Add(font);
        cmbFontFamily.Items.Add(font.Name);
    }
}

// 验证字体是否与 PDFsharp 兼容
private bool IsFontCompatibleWithPdfSharp(string fontName)
{
    try
    {
        using (var testDoc = new PdfDocument())
        {
            var testPage = testDoc.AddPage();
            using (var gfx = XGraphics.FromPdfPage(testPage))
            {
                var testFont = new XFont(fontName, 12, XFontStyleEx.Regular);
                gfx.DrawString("测试", testFont, XBrushes.Black, new XPoint(10, 10));
            }
        }
        return true;
    }
    catch
    {
        return false;
    }
}
```

**字体预览验证：**
```csharp
private void UpdateFontPreview()
{
    // 1. WinForms 预览
    lblFontPreview.Font = new Font(_selectedFont, 14F);
    lblFontPreview.Text = $"字体预览: {_selectedFont} - 中文测试 ABC 123";

    // 2. PDFsharp 兼容性验证
    bool pdfCompatible = TestPdfSharpFontRendering(_selectedFont);
    
    if (!pdfCompatible)
    {
        lblFontPreview.ForeColor = Color.Red;
        lblFontPreview.Text += " ⚠️ 此字体可能无法在PDF中正常显示";
    }
}
```

### 2. PDF 标识页插入改用 PDFsharp (PdfTools.cs)

**原因：**
- iText7 的 `TrueTypeCollection` API 对 .ttc 文件支持有问题
- PDFsharp 6.x 对系统字体支持更好，且是 MIT 许可证（完全免费）
- 项目已安装 PDFsharp 6.2.0

**修改内容：**
- 新增 `InsertIdentifierPageWithPdfSharp()` 方法
- 新增 `CreatePdfSharpFont()` 方法，**直接使用用户选择的字体，无回退**
- 新增 `DrawCenteredTextPdfSharp()` 方法，绘制居中文字
- 新增 `WrapTextPdfSharp()` 方法，处理文本换行

**字体加载逻辑（无回退）：**
```csharp
private static XFont CreatePdfSharpFont(float fontSize)
{
    // 直接使用用户选择的字体，不做回退
    string selectedFont = AppSettings.GetValue<string>("IdentifierPageFont") ?? "Microsoft YaHei";
    var font = new XFont(selectedFont, fontSize, XFontStyleEx.Regular);
    return font;
    
    // 如果失败，抛出异常提示用户在设置中选择兼容的字体
}
```

## 设计理念

### 快速加载 + 延迟验证

**加载策略：**
```
1. 快速加载所有系统字体（仅验证 WinForms 兼容性）
2. 用户选择字体后，异步验证 PDFsharp 兼容性
3. 如果不兼容，显示红色警告
4. 不阻塞UI，保持流畅体验
```

**验证流程：**
```
加载字体列表（快速） → 用户选择 → 异步验证 → 显示结果
```

### 优势

1. **快速加载**：字体列表立即显示，不需要等待
2. **所见即所见**：预览效果与 PDF 生成效果一致
3. **用户可控**：明确知道哪些字体可用
4. **避免困惑**：不会出现"选了A字体却显示B字体"的情况
5. **早期发现问题**：在设置阶段就发现字体兼容性问题
6. **明确错误提示**：如果字体不兼容，直接提示用户重新选择
7. **不阻塞UI**：验证在后台进行，界面保持响应

## 技术对比

| 特性 | iText7 | PDFsharp 6.x |
|------|--------|--------------|
| TTC 字体支持 | ❌ 需要特殊处理，容易出错 | ✅ 原生支持 |
| 系统字体 | ⚠️ 需要手动加载字体文件 | ✅ 直接使用字体名 |
| 中文支持 | ⚠️ 需要 IDENTITY_H 编码 | ✅ 自动处理 |
| 许可证 | AGPL（商业需付费） | MIT（完全免费） |
| 字体验证 | ❌ 无法提前验证 | ✅ 可以提前测试 |

## 数据流

```
系统字体列表
    ↓
快速加载（WinForms 验证）
    ↓
显示所有可用字体
    ↓
用户选择字体
    ↓
实时预览（WinForms 渲染）
    ↓
异步验证 PDFsharp 兼容性
    ↓
显示验证结果（黑色=兼容，红色=警告）
    ↓
保存字体名到 AppSettings
    ↓
PDF 处理时直接使用（无回退）
    ↓
生成标识页（效果与预览一致）
```

## 用户体验流程

1. **打开设置** → 看到所有兼容的系统字体
2. **选择字体** → 立即看到预览效果
3. **如果不兼容** → 显示红色警告，提示选择其他字体
4. **保存设置** → 字体名称被保存
5. **生成PDF** → 标识页使用选择的字体，效果与预览一致

## 错误处理

### 字体不兼容时
- **加载阶段**：不显示在列表中
- **预览阶段**：显示红色警告文字
- **生成阶段**：抛出明确的异常，提示用户重新选择

### 错误信息示例
```
无法创建字体，请在设置中选择兼容的字体。
错误: Font 'XXX' not found or not supported.
```

## 注意事项

1. **字体列表加载可能较慢**：因为需要验证每个字体的兼容性
2. **首次加载会有延迟**：建议显示加载提示
3. **字体名称区分大小写**：保存和使用时保持一致
4. **不兼容的字体不会显示**：用户只能看到可用的字体
5. **预览即所见**：预览效果就是最终 PDF 效果
