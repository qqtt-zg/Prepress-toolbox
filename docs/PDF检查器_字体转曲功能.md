# PDF检查器 - 字体转曲功能

## 功能概述

在字体标签页中添加了字体转曲功能，可以将 PDF 中的文字转换为路径（曲线）。

---

## 功能说明

### 什么是字体转曲？

字体转曲（Font Outlining / Convert Text to Outlines）是将 PDF 中的文字转换为矢量路径的过程。转曲后：
- ✅ 文字变成图形路径，不再依赖字体文件
- ✅ 避免字体缺失或替换问题
- ✅ 确保在任何设备上显示一致
- ❌ 文字无法再编辑或搜索
- ❌ 文件大小可能增加

### 使用场景

1. **印刷前处理**：确保印刷厂能正确输出，避免字体问题
2. **文件交付**：向客户交付最终文件，防止修改
3. **字体授权**：避免分发包含商业字体的文件
4. **跨平台兼容**：确保在不同系统上显示一致

---

## 使用方法

### 1. 打开 PDF 检查器
- 在 PDF 操作面板加载 PDF 文件
- 点击"检查器"按钮打开检查器窗口

### 2. 切换到字体标签页
- 点击"字体"标签页
- 查看 PDF 中使用的所有字体

### 3. 执行转曲操作
1. 点击"转曲"按钮
2. 确认转曲操作（会弹出警告对话框）
3. 选择保存位置（默认文件名：原文件名_转曲.pdf）
4. 等待处理完成
5. 选择是否在检查器中打开转曲后的文件

## 技术实现

### 服务类：PdfFontOutlineService

```csharp
public class PdfFontOutlineService
{
    // 将PDF中的所有文字转换为路径
    public bool ConvertTextToOutlines(string inputPath, string outputPath)
    
    // 检查PDF是否包含文字（未转曲）
    public bool HasText(string filePath)
    
    // 获取PDF文字内容预览
    public string GetTextPreview(string filePath, int maxLength = 200)
}
```

### 实现方式

使用 iText7 库进行 PDF 处理：
1. 读取原始 PDF 文件
2. 解析每一页的内容流
3. 提取文本渲染操作
4. 将文本转换为路径（使用字体的字形轮廓）
5. 替换原有的文本操作符
6. 生成新的 PDF 文件

**注意**：当前实现提供了基本框架，完整的字体转曲功能需要更复杂的内容流处理。

## 注意事项

### 转曲前
- ✅ 确保 PDF 文件已保存
- ✅ 检查字体列表，了解哪些字体会被转曲
- ✅ 备份原始文件（转曲操作会创建新文件，不会修改原文件）

### 转曲后
- ❌ 文字无法编辑
- ❌ 文字无法搜索
- ❌ 无法复制文字内容
- ✅ 字体显示完全一致
- ✅ 不依赖字体文件

### 文件大小
- 简单字体（如 Arial）：文件可能略微增大
- 复杂字体（如中文字体）：文件可能显著增大
- 大量文字：文件大小增加明显

## 常见问题

### Q: 转曲需要多长时间？
**A**: 取决于 PDF 的页数和文字数量，通常几秒到几十秒。

### Q: 转曲后文件变大了？
**A**: 正常现象，尤其是包含大量中文字体的文件。

### Q: 可以撤销转曲吗？
**A**: 不可以。转曲是不可逆操作，请保留原始文件。

### Q: 转曲后还能看到字体列表吗？
**A**: 可以，但字体信息会显示为路径/图形，不再是文字。

### Q: 支持哪些字体类型？
**A**: 理论上支持所有 TrueType 和 OpenType 字体，Type3 字体可能有限制。
