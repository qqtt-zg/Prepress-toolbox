# 事件分组节点交互方式优化

## 修改概述
优化了事件分组TreeView的节点交互方式，实现了当取消勾选分组节点时同时取消分组内所有子节点的功能，提供更直观的用户体验。

## 修改的文件

### EventGroupsTreeView.cs
**文件路径**: `src/WindowsFormsApp3/Controls/EventGroupsTreeView.cs`

#### 主要修改内容

1. **添加节点勾选事件绑定**
   - 在 `SetupTreeView()` 方法中添加了 `treeView.AfterCheck += TreeView_AfterCheck;`
   - 确保TreeView的勾选事件能够被正确处理

2. **实现TreeView_AfterCheck事件处理方法**
   - 添加了完整的节点勾选事件处理逻辑
   - 区分分组节点和项目节点的不同处理方式

3. **添加分组勾选处理方法**
   - `HandleGroupCheckChanged()`: 专门处理分组节点勾选状态变化
   - 当分组取消勾选时，强制取消所有子项目的勾选状态
   - 当分组勾选时，保持子项目原有状态（可配置策略）

4. **添加项目勾选处理方法**
   - `HandleItemCheckChanged()`: 处理单个项目节点的勾选状态变化
   - 提供了多种策略选择的扩展点

## 功能特性

### 🎯 核心功能
- **分组取消勾选联动**: 当取消勾选分组节点时，自动取消该分组内所有子项目的勾选状态
- **状态数据同步**: 确保TreeNode的勾选状态与TreeNodeData的IsEnabled属性保持同步
- **配置自动保存**: 勾选状态变化时自动触发配置保存事件

### 🔧 交互策略
当前实现采用以下交互策略：

1. **分组取消勾选**: 强制取消所有子项目勾选（确保一致性）
2. **分组勾选**: 保持子项目原有状态（让用户自主选择）
3. **项目独立勾选**: 允许项目独立于分组状态进行勾选

### 🎨 可扩展策略
代码中预留了多种策略选择：

```csharp
// 策略1：保持子项目原有状态（当前实现）
// 策略2：同时勾选所有子项目（可选）
// 策略3：询问用户是否同时勾选子项目（可选）
```

## 用户体验改进

### ✅ 改进前的问题
- 分组和子项目的勾选状态可能不一致
- 用户需要手动逐个取消子项目勾选
- 交互逻辑不够直观

### 🚀 改进后的优势
- **直观的层级联动**: 分组取消勾选时自动处理子项目
- **一致的状态管理**: 确保UI状态与数据状态同步
- **灵活的交互策略**: 支持不同的用户操作习惯
- **自动配置保存**: 减少用户手动保存的操作

## 技术实现细节

### 事件处理流程
```
用户勾选/取消勾选节点
    ↓
TreeView_AfterCheck 事件触发
    ↓
判断节点类型（分组 vs 项目）
    ↓
调用对应的处理方法
    ↓
更新节点数据状态
    ↓
触发配置保存事件
```

### 数据同步机制
- TreeNode.Checked ↔ TreeNodeData.IsEnabled
- 确保UI显示与内部数据模型保持一致
- 通过ConfigurationSaveRequested事件通知父控件保存配置
