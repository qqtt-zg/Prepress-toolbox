# 产品需求文档：排版功能模块（Imposition Module）

> **文档版本**: v1.0
> **状态**: Draft
> **日期**: 2026-05-09
> **负责人**: 大诚重命名工具开发团队

---

## 一、产品概述（What / Who / Why Now）

**是什么**：为"大诚重命名工具"添加专业级 PDF 拼版功能，覆盖 N-Up、骑马订、Cut & Stack、Step & Repeat、页面重排等印刷排版模式。

**给谁用**：中国不干胶标签/吊牌印刷厂的印前操作员，中小型印刷企业（10-50人规模），目前大量使用 CorelDRAW/AI 手动拼版的用户。

**为什么现在做**：市场调研发现￥1,000-5,000价格区间缺少专为中国标签印刷优化的轻量级拼版工具。火并 LabelImposer 和 PaSharp 需询价（数万元），Montax Imposer 功能不够专业化。项目已有骑马订（90%）、N-Up 引擎（40%）、裁切标记（80%）的基础，具备快速交付的条件。

---

## 二、问题陈述

中国中小型标签印刷厂的印前操作员每天需要将大量短版标签订单拼排到标准纸张上，以最大化材料利用率并减少印刷次数。当前工作流存在以下痛点：

1. **手动拼版效率低**：操作员在 CorelDRAW/AI 中手动排列页面，一个订单需要 10-30 分钟
2. **材料浪费严重**：缺乏自动布局优化，纸张利用率通常只有 60-75%
3. **重复劳动多**：相似订单（不同数量）需要反复手动调整布局参数
4. **专业工具门槛高**：Esko Phoenix、Kodak Preps 等专业工具价格昂贵（数万元起），学习曲线陡峭

> "我们每天处理 50+ 个短版标签订单，每个订单都要在 CorelDRAW 里手动拼版，光这一步就要花 2-3 个小时。" —— 某标签印刷厂印前主管

## 三、目标与非目标

### 目标（Goals）

| 目标 | 说明 |
|------|------|
| **G1. 覆盖核心拼版模式** | 支持 N-Up、骑马订、Cut & Stack、Step & Repeat、页面重排 5 种主流拼版模式 |
| **G2. 自动化布局优化** | 自动计算最优行列数，最大化纸张利用率（目标 >85%） |
| **G3. 中文原生体验** | 全中文界面，符合中国印刷行业术语习惯 |
| **G4. 集成现有工作流** | 与项目已有的文件重命名、Excel 数据导入功能无缝衔接 |
| **G5. 价格亲民** | 定价 ￥1,000-5,000 区间，远低于竞品数万元的价格 |

### 非目标（Non-Goals）

| 非目标 | 说明 |
|--------|------|
| **NG1. 包装盒展开图** | 不做 3D 折叠纸盒的展开图设计（Esko Phoenix 的领域） |
| **NG2. 印刷机直连** | 不对接 JDF/JMF 协议，不直接控制印刷设备 |
| **NG3. 色彩管理** | 不做 ICC 色彩配置文件管理（callas pdfToolbox 的领域） |
| **NG4. 云端/SaaS 模式** | 暂不做在线版本，专注桌面端 |
| **NG5. macOS 支持** | 仅支持 Windows 平台 |

## 五、用户故事

### Sprint 1：完善核心拼版

**US-1.1 N-Up 自动拼版**
> 作为印前操作员，我希望选择一个 PDF 文件后，系统自动计算最优的行列排列方式并生成拼版 PDF，以便快速完成常规的 N-Up 拼版任务。

验收标准：
- 支持选择纸张尺寸（A3/A4/A5/B4/SRA3/自定义）
- 支持自动计算最优行列数（显示空间利用率）
- 支持手动指定行列数
- 支持设置间距（Gutter）和出血（Bleed）
- 支持自动旋转优化（提高利用率）
- 生成的 PDF 页面尺寸正确，内容位置准确
- 支持一式 N 联功能

**US-1.2 裁切标记集成**
> 作为印前操作员，我希望在生成拼版 PDF 时可以选择添加裁切标记，以便印刷后能准确裁切。

验收标准：
- 拼版生成时可选择是否添加裁切标记
- 支持配置标记长度、偏移距离、线宽
- 支持选择页面范围（全部/当前/自定义）
- 裁切标记绘制在页面出血区域外，不遮挡内容
- 支持套准标记（同心圆+十字线）

**US-1.3 页面重排（Shuffle）**
> 作为印前操作员，我希望将 PDF 页面按指定顺序重新排列，以便满足特定的装订或裁切需求。

验收标准：
- 支持列式排列（Column Shuffle）
- 支持自定义页序（如 1,3,5,2,4,6）
- 支持页面范围选择
- 生成的 PDF 页序正确

### Sprint 2：标签印刷核心功能

**US-2.1 Cut & Stack 排版**
> 作为标签印刷操作员，我希望使用 Cut & Stack 模式拼版，以便在裁切后能按顺序堆叠成品。

验收标准：
- 支持标准 Cut & Stack 模式（如 111/222/333 → 裁切堆叠为 123/123/123）
- 支持指定每堆数量
- 自动计算最优布局

**US-2.2 Step & Repeat 增强**
> 作为标签印刷操作员，我希望精确控制标签在纸张上的排列间距，以便满足模切和后道工序要求。

验收标准：
- 支持行列间距独立设置
- 支持出血重叠设置
- 支持自定义起始位置

**US-2.3 混拼支持**
> 作为标签印刷操作员，我希望将不同尺寸的 PDF 混合排列在同一张纸上，以便最大化材料利用率。

验收标准：
- 支持选择多个不同尺寸的 PDF 文件
- 自动计算最优混排方案
- 显示每个文件的位置和旋转信息

**US-2.4 DXF 刀线导出**
> 作为标签印刷操作员，我希望将拼版布局导出为 DXF 文件，以便数码模切机识别刀线位置。

验收标准：
- 导出标准 DXF 格式文件
- 包含裁切线和标记位置
- 可被主流数码模切软件识别

### Sprint 3：自动化与效率提升

**US-3.1 热文件夹监控**
> 作为印前操作员，我希望设置一个监控文件夹，当有新 PDF 文件放入时自动触发拼版，以便实现无人值守的批量处理。

验收标准：
- 支持指定监控文件夹路径
- 支持配置拼版模板（参数预设）
- 新文件放入后自动触发拼版
- 输出到指定结果文件夹
- 支持处理日志和错误通知

**US-3.2 批量拼版输出**
> 作为印前操作员，我希望一次性选择多个 PDF 文件进行批量拼版，以便提高处理效率。

验收标准：
- 支持多文件选择
- 统一应用相同的拼版参数
- 显示批量处理进度
- 支持失败重试

**US-3.3 拼版模板保存/加载**
> 作为印前操作员，我希望将常用的拼版参数保存为模板，以便快速复用。

验收标准：
- 支持保存当前参数为模板
- 支持加载已有模板
- 支持模板命名和删除
- 模板持久化存储

**US-3.4 PDF 预检**
> 作为印前操作员，我希望在拼版前检查 PDF 文件的质量问题，以便避免印刷事故。

验收标准：
- 检测字体缺失
- 检测颜色空间（RGB/CMYK）
- 检测出血不足
- 检测页面尺寸不一致
- 提供问题报告和建议

## 六、功能范围与里程碑

### Phase 1：核心拼版（Sprint 1，2-3 周）

```
[✓] 骑马订拼版（已完成 90%）
[→] N-Up 参数面板 + PDF 生成
[→] 裁切标记集成
[→] 页面重排 Shuffle
```

关键交付物：
- N-Up 拼版端到端可用
- 裁切标记可在拼版流程中选择性添加
- 页面重排功能可用

### Phase 2：标签专用功能（Sprint 2，3-4 周）

```
[ ] Cut & Stack 排版
[ ] Step & Repeat 增强
[ ] 混拼支持
[ ] DXF 刀线导出
```

关键交付物：
- Cut & Stack 和 Step & Repeat 覆盖标签印刷主要场景
- 混拼功能可用
- DXF 导出对接数码模切机

### Phase 3：自动化（Sprint 3，3-4 周）

```
[ ] 热文件夹监控
[ ] 批量拼版输出
[ ] 拼版模板系统
[ ] PDF 预检
```

关键交付物：
- 热文件夹自动化工作流
- 批量处理能力
- 模板复用机制

### 设计模式

- **MVP 模式**：Panel + Presenter + View 接口
- **服务定位器**：ServiceLocator 管理服务生命周期
- **事件总线**：IEventBus 组件间解耦通信
- **模板方法**：ImpositionBase 定义拼版流程骨架，子类实现具体逻辑

## 附录 B：术语表
