# 标识页字体功能测试指南

## 测试环境
- Windows 系统
- .NET Framework 4.8
- PDFsharp 6.2.0

## 测试步骤

### 1. 字体选择测试

**步骤：**
1. 打开设置界面
2. 进入"标识页字体"设置
3. 查看字体下拉列表

**预期结果：**
- 列表显示系统所有已安装字体
- 中文字体（如微软雅黑、黑体、宋体等）显示在列表顶部
- 字体预览区域显示："字体预览: [字体名] - 中文测试 ABC 123"

### 2. 字体保存测试

**步骤：**
1. 选择一个中文字体（如"楷体"）
2. 点击保存按钮
3. 关闭并重新打开设置界面

**预期结果：**
- 重新打开后，之前选择的字体仍然被选中
- 字体预览正确显示

### 3. 标识页生成测试

**步骤：**
1. 在设置中选择字体（如"微软雅黑"）
2. 勾选"标识页"选项
3. 选择一个 PDF 文件进行处理
4. 设置标识页内容（如"测试文档-中文English123"）
5. 执行文件重命名/处理

**预期结果：**
- PDF 第一页插入标识页
- 标识页显示白色背景
- 文字使用选择的字体
- 中文、英文、数字都正常显示
- 文字水平和垂直居中

### 4. 长文本换行测试

**步骤：**
1. 设置较长的标识页内容（如"这是一个很长的测试文本用来验证自动换行功能是否正常工作This is a long test text"）
2. 处理 PDF 文件

**预期结果：**
- 文本自动换行
- 每行文字不超出页面边界
- 多行文字垂直居中
- 行间距合理

### 5. 不同字体对比测试

**测试字体列表：**
- 微软雅黑 (Microsoft YaHei)
- 黑体 (SimHei)
- 宋体 (SimSun)
- 楷体 (KaiTi)
- 仿宋 (FangSong)

**步骤：**
1. 依次选择不同字体
2. 使用相同的标识页内容
3. 生成多个 PDF 文件

**预期结果：**
- 所有字体都能正常显示中文
- 不同字体有明显的视觉差异
- 没有乱码或方框

### 6. 页面旋转测试

**步骤：**
1. 使用一个包含旋转页面的 PDF 文件
2. 插入标识页

**预期结果：**
- 标识页的旋转角度与原文档第一页一致
- 文字方向正确

### 7. 日志检查

**查看日志文件：** `app_2026-01-12.log`

**关键日志信息：**
```
尝试加载字体: [字体名]
✓ 成功创建系统字体: [字体名]
使用字体: [字体名]
PDFsharp绘制文字完成，共X行
PDFsharp标识页插入成功完成
```

## 常见问题排查

### 问题1：字体显示为方框或乱码

**可能原因：**
- 系统缺少选择的字体
- 字体文件损坏

**解决方法：**
1. 查看日志，确认实际使用的字体
2. 尝试选择其他字体
3. 检查系统字体是否正常安装

### 问题2：标识页插入失败

**可能原因：**
- PDF 文件被占用
- 磁盘空间不足
- PDF 文件损坏

**解决方法：**
1. 查看日志中的错误信息
2. 确保 PDF 文件未被其他程序打开
3. 检查磁盘空间

### 问题3：文字不居中

**可能原因：**
- 页面尺寸异常
- 旋转角度处理问题

**解决方法：**
1. 查看日志中的页面尺寸信息
2. 尝试使用标准尺寸的 PDF 文件测试

## 性能测试

**测试场景：**
- 小文件（1-10页）
- 中等文件（10-100页）
- 大文件（100+页）

**测试指标：**
- 标识页插入时间
- 内存占用
- 文件大小变化

**预期性能：**
- 小文件：< 1秒
- 中等文件：< 3秒
- 大文件：< 10秒
