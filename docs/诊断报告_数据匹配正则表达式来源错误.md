# 诊断报告：数据匹配正则表达式来源错误（修订版）

## 问题描述

当前在导入表格进行数据匹配时，正则表达式的使用逻辑存在问题：

### 正确的逻辑应该是：

1. **有保留分组前缀的文件**：
   - 当原文件名有事件分组的分组标志（如 `&ID-`, `&MT-` 等）
   - 且事件分组设置了保留分组功能时
   - 导入表格后需要数据匹配时，使用的正则结果需要从原文件名上提取
   - **这个功能当前是正常的，不应被影响**

2. **没有保留分组前缀的文件**：
   - 当原文件名没有事件分组的分组标志时
   - 在导入表格需要数据匹配时
   - 需要使用 `ExcelImportForm.cs` 的 `cmbRegex2` 控件选择的正则表达式去完成数据匹配
   - **这个功能当前有问题**

### 当前实现的问题：

原来的修复逻辑顺序错误：
1. 先使用 `ExtractRegexResultForMatching` 提取正则结果（使用 ExcelImportService 的正则表达式）
2. 然后检查是否有保留分组前缀，如果有则覆盖

这导致：
- 当文件没有保留分组前缀时，如果 `ExtractRegexResultForMatching` 返回空值，会直接跳过匹配
- 逻辑不清晰，容易出错

## 问题分析

### 1. 数据匹配流程

在 `FileRenamePanelPresenter.cs` 中，数据匹配的流程如下：

```csharp

// 行: 1233-1320

{
    // ...
    foreach (var fileInfo in fileList)
    {

        {
            continue;
        }

        string regexResultForMatching = fileInfo.RegexResult;
        
        // 检测并使用保留的正则结果（如果有）
        // ...

        // ...
    }

```

### 2. 正则结果的提取

`fileInfo.RegexResult` 是在文件处理时通过 `ExtractRegexResult` 方法提取的：

```csharp
// 文件: src/WindowsFormsApp3/Presenters/FileRenamePanelPresenter.cs
// 行: 1970-2015

public string ExtractRegexResult(string fileName)
{
    try
    {
        // ❌ 问题所在：使用的是 FileRenamePanel 的 _cmbRegex 控件
        var pattern = _view.SelectedRegexValue;
        _logger?.LogInformation($"[ExtractRegexResult] 开始提取正则结果 - 文件名: {fileName}, 正则: {pattern}");
        
        if (string.IsNullOrEmpty(pattern))
        {
            _logger?.LogWarning($"[ExtractRegexResult] 正则表达式为空");
            return null;
        }

        var match = Regex.Match(fileName, pattern);
        if (match.Success)
        {
            // 提取匹配结果
            string result;
            if (match.Groups.Count > 1 && match.Groups[1].Success)
            {
                result = match.Groups[1].Value;
            }
            else
            {
                result = match.Value;
            }
            return result;
        }

        return null;
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, $"提取正则结果失败: {fileName}");
        return null;
    }
}
```

### 3. SelectedRegexValue 属性的实现

`_view.SelectedRegexValue` 属性在 `FileRenamePanel.cs` 中实现：

```csharp
// 文件: src/WindowsFormsApp3/Forms/Panels/FileRenamePanel.cs
// 行: 1117-1156

public string SelectedRegexValue
{
    get
    {
        // ❌ 问题所在：从 FileRenamePanel 的 _cmbRegex 控件获取正则表达式
        try
        {
            var selectedName = SelectedRegexPattern;
            if (string.IsNullOrEmpty(selectedName))
            {
                return null;
            }

            // 从AppSettings加载正则表达式字典
            var regexStr = AppSettings.Instance.RegexPatterns;
            if (string.IsNullOrEmpty(regexStr))
            {
                return null;
            }

            // 解析 "名称=表达式|名称=表达式" 格式
            var patterns = regexStr.Split(new[] { '|' }, StringSplitOptions.RemoveEmptyEntries);
            foreach (var pattern in patterns)
            {
                var parts = pattern.Split(new[] { '=' }, 2);
                if (parts.Length == 2 && parts[0] == selectedName)
                {
                    return parts[1]; // 返回正则表达式值
                }
            }

            return null;
        }
        catch (Exception ex)
        {
            LogHelper.Error($"获取正则表达式失败: {ex.Message}", ex);
            return null;
        }
    }
}

public string SelectedRegexPattern
{
    get
    {
        // ❌ 问题所在：从 FileRenamePanel 的 _cmbRegex 控件获取选中的模式名称
        if (_cmbRegex != null && _cmbRegex.SelectedIndex >= 0 && _cmbRegex.Items.Count > 0)
        {
            return _cmbRegex.Items[_cmbRegex.SelectedIndex]?.ToString();
        }
        return null;
    }
}
```

## 问题根源

**核心问题**：在数据匹配时，系统使用的是 `FileRenamePanel._cmbRegex` 控件选择的正则表达式，而不是 `ExcelImportForm.cmbRegex2` 控件选择的正则表达式。

这导致：
1. 用户在 `ExcelImportForm` 中选择的正则表达式（用于数据匹配）被忽略
2. 系统错误地使用 `FileRenamePanel` 中选择的正则表达式（用于文件名处理）来提取文件名中的正则结果
3. 数据匹配结果不符合预期

## 解决方案

### 方案 1：在 ExcelImportService 中存储正则表达式

修改 `ExcelImportService` 接口和实现，添加正则表达式属性：

```csharp

    string SelectedRegexPattern { get; set; }

    // ...
```

### 方案 2：在数据匹配时使用 ExcelImportForm 的正则表达式

修改 `FileRenamePanelPresenter.ExtractRegexResult` 方法，在数据匹配场景下使用 `ExcelImportService` 中存储的正则表达式：

```csharp
// 文件: src/WindowsFormsApp3/Presenters/FileRenamePanelPresenter.cs

public string ExtractRegexResult(string fileName, bool useExcelRegex = false)
{
    try
    {
        // ✅ 修复：根据场景选择正确的正则表达式
        string pattern;
        if (useExcelRegex && _excelImportService != null)
        {
            // 数据匹配场景：使用 ExcelImportForm 的正则表达式
            pattern = _excelImportService.SelectedRegexPattern;
        }
        else
        {
            // 文件名处理场景：使用 FileRenamePanel 的正则表达式
            pattern = _view.SelectedRegexValue;
        }
        
        _logger?.LogInformation($"[ExtractRegexResult] 开始提取正则结果 - 文件名: {fileName}, 正则: {pattern}, 使用Excel正则: {useExcelRegex}");
        
        if (string.IsNullOrEmpty(pattern))
        {
            _logger?.LogWarning($"[ExtractRegexResult] 正则表达式为空");
            return null;
        }

        var match = Regex.Match(fileName, pattern);
        if (match.Success)
        {
            string result;
            if (match.Groups.Count > 1 && match.Groups[1].Success)
            {
                result = match.Groups[1].Value;
            }
            else
            {
                result = match.Value;
            }
            return result;
        }

        return null;
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, $"提取正则结果失败: {fileName}");
        return null;
    }
}
```

### 方案 3：在 MatchExcelData 中重新提取正则结果

修改 `MatchExcelData` 方法，在数据匹配时使用 `ExcelImportForm` 的正则表达式重新提取正则结果：

```csharp
// 文件: src/WindowsFormsApp3/Presenters/FileRenamePanelPresenter.cs

public void MatchExcelData()
{
    try
    {
        var excelData = _view.ExcelData;
        var fileList = _view.FileList;

        if (excelData == null || fileList == null)
        {
            _logger?.LogDebug("MatchExcelData: Excel数据或文件列表为空，跳过匹配");
            return;
        }

        // 获取列索引
        int searchColumnIndex = _excelImportService.SearchColumnIndex;
        int returnColumnIndex = _excelImportService.ReturnColumnIndex;
        int serialColumnIndex = _excelImportService.SerialColumnIndex;

        if (searchColumnIndex < 0 || returnColumnIndex < 0)
        {
            _logger?.LogDebug($"MatchExcelData: 列索引无效 - 搜索列:{searchColumnIndex}, 返回列:{returnColumnIndex}");
            return;
        }

        _logger?.LogDebug($"MatchExcelData: 开始匹配 - 搜索列:{searchColumnIndex}, 返回列:{returnColumnIndex}, 序号列:{serialColumnIndex}");

        int matchedCount = 0;

        foreach (var fileInfo in fileList)
        {
            // ✅ 修复：使用 ExcelImportForm 的正则表达式重新提取正则结果
            string regexResultForMatching = ExtractRegexResult(fileInfo.OriginalName ?? "", useExcelRegex: true);
            
            if (string.IsNullOrEmpty(regexResultForMatching))
            {
                continue;
            }

            // 检测并使用保留的正则结果（如果有）
            if (!string.IsNullOrEmpty(fileInfo.OriginalName))
            {
                string originalNameWithoutExt = Path.GetFileNameWithoutExtension(fileInfo.OriginalName);
                
                if (System.Text.RegularExpressions.Regex.IsMatch(originalNameWithoutExt, @"&[A-Z]+-"))
                {
                    _logger?.LogDebug($"检测到保留分组前缀，尝试提取保留的正则结果: '{originalNameWithoutExt}'");
                    
                    var preserveGroupConfig = GetPreserveGroupConfig();
                    
                    if (preserveGroupConfig != null && preserveGroupConfig.Count > 0)
                    {
                        var currentPrefixes = GetCurrentPrefixes();
                        
                        var tempComponents = new Models.FileNameComponents
                        {
                            OriginalFileName = originalNameWithoutExt,
                            PreserveGroupConfig = preserveGroupConfig,
                            Prefixes = currentPrefixes
                        };
                        
                        var preserveData = tempComponents.ExtractPreserveGroupData(originalNameWithoutExt);
                        
                        if (preserveData != null && preserveData.ContainsKey("正则结果"))
                        {
                            string preservedRegexResult = preserveData["正则结果"];
                            if (!string.IsNullOrEmpty(preservedRegexResult))
                            {
                                regexResultForMatching = preservedRegexResult;
                                _logger?.LogInformation($"✅ 使用保留的正则结果进行匹配: '{regexResultForMatching}' (重新提取的结果: '{regexResultForMatching}', 文件: '{fileInfo.OriginalName}')");
                            }
                        }
                    }
                }
            }

            var matchData = MatchExcelData(regexResultForMatching);

            if (matchData != null && matchData.HasMatch)
            {
                // 更新文件信息
                fileInfo.Quantity = matchData.Quantity;
                fileInfo.SerialNumber = matchData.SerialNumber;
                if (!string.IsNullOrEmpty(matchData.Material))
                {
                    fileInfo.Material = matchData.Material;
                }
                if (!string.IsNullOrEmpty(matchData.CompositeColumn))
                {
                    fileInfo.CompositeColumn = matchData.CompositeColumn;
                    _logger?.LogDebug($"设置列组合数据: 文件='{fileInfo.OriginalName}', 列组合='{matchData.CompositeColumn}'");
                }
                matchedCount++;

                _logger?.LogDebug($"匹配成功: RegexResult='{regexResultForMatching}' -> 数量='{matchData.Quantity}', 序号='{matchData.SerialNumber}', 材料='{matchData.Material}', 列组合='{matchData.CompositeColumn}'");
            }
        }

        _view.RefreshFileTable();
        _view.UpdateStatus($"匹配完成，成功匹配 {matchedCount} 个文件");
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, "匹配Excel数据失败");
        _view.ShowError($"匹配Excel数据失败: {ex.Message}");
    }
}
```

## 推荐方案

**推荐使用方案 1 + 方案 3 的组合**：

1. 在 `ExcelImportService` 中添加 `SelectedRegexPattern` 属性，用于存储 `ExcelImportForm` 选择的正则表达式
2. 在 `ExcelImportForm.BtnOK_Click` 中，将选择的正则表达式保存到 `ExcelImportService`
3. 在 `FileRenamePanelPresenter.MatchExcelData` 中，使用 `ExcelImportService.SelectedRegexPattern` 重新提取正则结果进行数据匹配

这样可以确保：
- 数据匹配使用的是 `ExcelImportForm` 中选择的正则表达式
- 文件名处理使用的是 `FileRenamePanel` 中选择的正则表达式
- 两者互不干扰，各司其职
