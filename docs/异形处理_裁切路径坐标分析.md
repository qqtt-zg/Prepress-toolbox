# 异形处理 - 裁切路径坐标不需要偏移的原因

## 根本原因

PDF 内容流中的 cm 变换在 q/Q 保护对内，不影响路径坐标。

## 详细分析

模板页内容流结构：
`
q
0.99983 0 0 0.99975 10.67605 -0.83744 cm   <- cm 在 q/Q 保护对内
/RelativeColorimetric ri
1 0 0 RG
0.0283464 w
1 J
1 j
486.1191 241.7051 m                         <- 路径坐标
...path operators...
S
Q
`

### 为什么 cm 不影响路径坐标

1. cm 在 q/Q 保护对内：q 保存图形状态，cm 修改 CTM，路径操作符使用修改后的 CTM，Q 恢复图形状态
2. ExtractAndConvertCutPath 读取的是内容流的原始字节，路径坐标已经是页面可视区域（CropBox）坐标系中的值
3. ConvertLineCoords 中累加的 cmTx/cmTy 实际上是多余的（但因为缩放因子约等于1.0，误差可忽略）

### 为什么不需要 ConcatMatrix

路径坐标在 CropBox 坐标系中的位置已经正确。
ExtractAndConvertCutPath 提取的坐标直接就是页面可视区域空间中的值，不需要任何变换。

### ConcatMatrix 为什么会出错

之前尝试的偏移公式 originDiff + centerX - pathOffsetX 实际上是在对抗坐标系本身的正确性：
- pathOffsetX = pathCenter - templateCenter（路径在模板中的偏移）
- originDiff = contentOrigin - templateOrigin（页面原点差）
- 两者相消为零，导致 ConcatMatrix 输出近零值
- 加上 ConcatMatrix 反而把路径推到错误位置

### 结论

| 操作 | 是否需要 | 原因 |
|---|---|---|
| ConcatMatrix 偏移 | 不需要 | 坐标已在 CropBox 空间中 |
| pathOffset 修正 | 不需要 | 同上 |
| ConvertLineCoords | 保留 | 保持路径坐标原样传递 |
| centerX/Y 居中 | 仅出血线 | 裁切路径不需要 |

## 出血线 ConcatMatrix 问题（待验证）

### 现状

出血线仍然使用 ConcatMatrix 偏移：
`csharp
float bleedOffsetX = (float)(currentPageSize.GetLeft() - templateCropLeft + centerX);
float bleedOffsetY = (float)(currentPageSize.GetBottom() - templateCropBottom + centerY);
bleedCanvas.ConcatMatrix(1, 0, 0, 1, bleedOffsetX, bleedOffsetY);
`

### 问题

ConcatMatrix 在 BDC（BeginDesignatedContent）标记内容内可能不被 Adobe Acrobat 应用。这与裁切路径遇到的问题相同。

### 当前效果

出血线目前效果正确（绿色矩形画在页面边界上）。可能原因：
1. ConcatMatrix 对出血线恰好生效（与裁切路径不同的 BDC 结构）
2. 或者出血线位置恰好在正确位置（与裁切路径不同，出血线用的是当前页 CropBox 坐标 + 偏移）

### 如果后续出血线位置出错

修改方案：移除 ConcatMatrix，直接使用当前页 CropBox 坐标：
`csharp
// 替换为：
bleedCanvas.Rectangle(
    (float)currentPageSize.GetLeft(),
    (float)currentPageSize.GetBottom(),
    (float)currentPageSize.GetWidth(),
    (float)currentPageSize.GetHeight());
// 无需 ConcatMatrix
`

原因：出血线应该画在当前页的 CropBox 边界上，而不是模板页的 CropBox 坐标再用 ConcatMatrix 偏移。坐标已经在正确的空间中。
