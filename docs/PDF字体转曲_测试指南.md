# PDF 字体转曲功能 - 测试指南

## ✅ 前置条件

- [x] Ghostscript 已安装（版本：10.06.0）
- [x] 应用程序已编译成功
- [x] 标签页显示正常

---

## 🧪 测试步骤

### 1. 启动应用程序

运行编译后的应用程序：
```
F:\编程项目\Prepress-toolbox\src\WindowsFormsApp3\bin\Debug\net48\win-x64\大诚重命名工具.exe
```

### 2. 加载 PDF 文件

1. 在主界面点击"PDF操作"面板
2. 点击"选择文件"按钮
3. 选择一个包含文字的 PDF 文件
4. 文件加载成功后，PDF 预览应该显示

### 3. 打开 PDF 检查器

1. 点击工具栏上的"检查器"按钮（带放大镜图标）
2. 检查器窗口应该弹出
3. 确认可以看到 4 个标签页：
   - 当前页面
   - 所有页面
   - 问题
   - 字体

### 4. 查看字体信息

1. 点击"字体"标签页
2. 应该看到：
   - 顶部工具栏：[转曲] 按钮 + 说明文字
   - 下方表格：显示 PDF 中使用的所有字体
3. 检查字体列表是否正确显示：
   - 状态（✓ 或 ⚠）
   - 字体名称
   - 类型
   - 嵌入状态
   - 使用页面
   - 问题描述

### 5. 测试字体转曲功能

#### 步骤 1：点击转曲按钮
1. 点击"转曲"按钮（带闪电图标）
2. 应该弹出确认对话框

#### 步骤 2：确认对话框
对话框内容应该是：
```
标题：确认转曲
内容：字体转曲后将无法编辑文字内容，是否继续？

      注意：此操作会创建新文件，不会修改原文件。

按钮：[确定] [取消]
```

3. 点击"确定"继续

#### 步骤 3：保存对话框
1. 应该弹出文件保存对话框
2. 检查默认设置：
   - 文件类型：PDF文件 (*.pdf)
   - 默认文件名：原文件名_转曲.pdf
   - 初始目录：原文件所在目录
3. 选择保存位置（或使用默认）
4. 点击"保存"

#### 步骤 4：等待处理
1. 应用程序开始执行转曲
2. 等待处理完成（通常几秒到几十秒）
3. 观察日志输出（如果有）

#### 步骤 5：成功提示
应该弹出成功提示对话框：
```
标题：转曲成功
内容：文件已保存到：
      C:\path\to\file_转曲.pdf
```

#### 步骤 6：打开新文件
1. 应该弹出询问对话框：
```
标题：打开文件
内容：是否在检查器中打开转曲后的文件？

按钮：[打开] [取消]
```

2. 点击"打开"
3. 检查器应该加载转曲后的 PDF 文件

### 6. 验证转曲结果

#### 检查 A：文件是否创建
1. 打开文件管理器
2. 导航到保存位置
3. 确认新文件存在：`原文件名_转曲.pdf`
4. 检查文件大小（通常会比原文件大）

#### 检查 B：文字是否转曲
1. 使用 PDF 阅读器打开转曲后的文件
2. 尝试选择文字：
   - ❌ 应该无法选择文字
   - ❌ 应该无法复制文字
   - ❌ 应该无法搜索文字
3. 检查视觉效果：
   - ✅ 文字显示应该与原文件一致
   - ✅ 字体样式应该保持不变
   - ✅ 布局应该完全相同

#### 检查 C：字体信息
1. 在检查器中查看转曲后文件的字体标签页
2. 字体列表应该：
   - 显示为空（没有字体）
   - 或显示为图形/路径

**预期结果**：
- 保存失败
- 显示错误提示
- 原文件未被修改

**测试文件**：使用大型 PDF（100+ 页）

**预期结果**：
- 转曲时间较长但能完成
- 进度提示清晰
- 不会卡死或崩溃

## 🐛 调试指南（v1.3.4 更新）

### 按钮状态问题排查

如果转曲按钮显示为灰色（禁用状态），请按以下步骤排查：

#### 步骤 1: 检查日志

查看应用程序目录下的最新日志文件（`app_YYYY-MM-DD.log`），搜索以下关键字：

```
[PdfInspectorControl] 转曲按钮已创建
[PdfInspectorControl] LoadPdf 开始
[PdfInspectorControl] 启用转曲按钮
[PdfInspectorControl] 转曲按钮已启用
```

#### 步骤 2: 正常日志输出

正常情况下应该看到：

```
[DEBUG] [PdfInspectorControl] 开始创建字体标签页
[DEBUG] [PdfInspectorControl] 转曲按钮已创建: Enabled=False, Visible=True
[DEBUG] [PdfInspectorControl] 字体标签页创建完成: Text=字体, Name=fontsTab, HasControls=True
[DEBUG] [PdfInspectorControl] LoadPdf 开始: C:\path\to\file.pdf, 页码: 1
[DEBUG] [PdfInspectorControl] _outlineButton 是否为 null: False
[DEBUG] [PdfInspectorControl] PDF信息加载成功，开始更新显示
[DEBUG] [PdfInspectorControl] 启用转曲按钮，当前状态: Enabled=False, Visible=True
[DEBUG] [PdfInspectorControl] 转曲按钮已启用(直接): Enabled=True
```

#### 步骤 3: 常见问题

**问题 1: 按钮未创建**
```
[WARN] [PdfInspectorControl] _outlineButton 为 null，无法启用
```
- 原因：CreateFontsTab 方法执行失败
- 解决：检查是否有其他错误日志

**问题 2: LoadPdf 未被调用**
- 日志中没有 "LoadPdf 开始" 消息
- 原因：检查器打开时未加载 PDF
- 解决：先加载 PDF，再打开检查器

**问题 3: LoadPdf 抛出异常**
```
[ERROR] [PdfInspectorControl] 加载PDF检查器失败: ...
```
- 原因：PDF 文件损坏或服务初始化失败
- 解决：尝试使用其他 PDF 文件

### 技术改进（v1.3.4）

1. **UI 线程安全**
   - 使用 `Invoke` 确保按钮启用操作在 UI 线程上执行
   - 避免跨线程操作导致的问题

2. **文件路径跟踪**
   - 添加 `_loadedFilePath` 字段
   - 确保转曲功能可以正确获取文件路径

3. **详细日志**
   - 在关键位置添加调试日志
   - 便于追踪问题和诊断
