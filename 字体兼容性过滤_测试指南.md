# 字体兼容性过滤功能 - 测试指南

## 功能概述

字体下拉列表现在只显示与 PDFsharp 兼容的字体，自动过滤掉不兼容的字体。

## 测试前准备

### 1. 确认代码已编译

```bash
# 编译项目
dotnet build src/WindowsFormsApp3/WindowsFormsApp3.csproj
```

### 2. 确认依赖包

确认以下 NuGet 包已安装：
- ✅ PDFsharp 6.2.0
- ✅ PdfiumViewer.Native.x86_64.v8-xfa 2018.4.8.256

### 3. 确认字体初始化

确认 `Program.cs` 中调用了：
```csharp
PdfSharpFontInitializer.Initialize();
```

## 测试步骤

### 测试 1: 字体列表加载

**目标：** 验证字体列表只显示兼容的字体

**步骤：**
1. 启动应用程序
2. 打开"设置" → "标识页字体"
3. 观察字体下拉列表的加载过程

**预期结果：**
```
初始状态：
  提示文本: "正在加载兼容的字体列表，请稍候..."
  提示颜色: 蓝色

加载中：
  提示文本: "正在加载兼容的字体列表... (50/200，已找到 35 个兼容字体)"
  提示颜色: 蓝色
  进度更新: 每验证 10 个字体更新一次

加载完成：
  提示文本: "已加载 X 个兼容字体（共 Y 个系统字体）"
  提示颜色: 绿色
  字体列表: 只包含兼容的字体
```

**验证点：**
- ✅ 加载过程不阻塞 UI（窗口可以移动、最小化）
- ✅ 显示实时进度
- ✅ 最终只显示兼容的字体
- ✅ 中文字体优先显示（微软雅黑、宋体等在前面）

**目标：** 验证列表中的所有字体都是兼容的

验证完成后：

## 常见问题排查

### 问题 1: 字体列表为空

**可能原因：**
- 所有字体都不兼容
- PDFsharp 初始化失败
- 字体解析器未正确设置

**排查步骤：**
1. 查看日志文件
2. 检查 `PdfSharpFontInitializer.Initialize()` 是否被调用
3. 检查 `SystemFontResolver` 是否正确加载

### 问题 2: 加载时间过长

**可能原因：**
- 系统字体数量过多
- 验证逻辑太复杂

**排查步骤：**
1. 查看日志中的字体数量
2. 检查 `TestFontCompatibilityQuick` 方法的性能
3. 考虑添加缓存机制

### 问题 3: 某些字体显示不兼容

**可能原因：**
- 字体文件格式问题
- PDFsharp 版本限制
- 字体文件损坏

**排查步骤：**
1. 查看日志中的错误信息
2. 检查字体文件是否存在
3. 尝试重新安装字体

### 问题 4: PDF 预览不显示

**可能原因：**
- PdfiumViewer 未正确安装
- 临时文件创建失败
- 图像渲染失败

**排查步骤：**
1. 查看日志中的错误信息
2. 检查 PdfiumViewer 包是否正确引用
3. 检查临时文件夹权限
