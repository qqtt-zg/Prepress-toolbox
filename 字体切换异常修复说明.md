# 字体切换异常修复说明

## 问题描述

### 错误信息
```
未经处理的异常：System.NullReferenceException
HResult=0x80004003
Message=未将对象引用设置到对象的实例。
Source=PdfSharp
StackTrace:
在 PdfSharp.Fonts.OpenType.OpenTypeFontFace.CetOrCreateFrom(XFontSource fontSource)
```

### 触发条件
- 在字体下拉列表（cmbFontFamily）中切换到某些特定字体时
- 异常发生在 PDFsharp 内部的字体解析过程中

### 根本原因

1. **PDFsharp 内部空引用**
   - PDFsharp 在解析某些字体文件时遇到格式问题
   - `OpenTypeFontFace.CetOrCreateFrom` 方法中的空引用
   - 某些字体文件格式不完全符合 OpenType 规范

2. **缺少异常处理**
   - 字体测试代码没有捕获 `NullReferenceException`
   - 异常传播到 UI 线程导致程序崩溃
   - 没有对字体文件进行预验证

3. **异步操作异常处理不完善**
   - 异步测试任务中的异常没有被正确捕获
   - UI 更新时可能控件已释放

## 解决方案

### 1. 增强异常处理（SettingsFontTextControl.cs）

#### A. 字体测试方法异常处理

**改进前：**

**改进后：**

### 2. 字体文件验证（SystemFontResolver.cs）

#### A. 添加字体文件验证

**新增方法：**
```csharp
/// <summary>
/// 验证字体文件数据（基本检查）
/// </summary>
private bool ValidateFontData(byte[] fontData, string fontFilePath)
{
    try
    {
        if (fontData == null || fontData.Length < 12)
        {
            LogHelper.Debug($"字体文件太小: {fontFilePath}");
            return false;
        }

        // 检查文件头标识
        string extension = Path.GetExtension(fontFilePath).ToLower();
        
        if (extension == ".ttf" || extension == ".otf")
        {
            // TrueType/OpenType 字体文件头检查
            uint signature = (uint)((fontData[0] << 24) | (fontData[1] << 16) | 
                                   (fontData[2] << 8) | fontData[3]);
            
            bool isValidTTF = signature == 0x00010000 || 
                             signature == 0x74727565 || // "true"
                             signature == 0x74797031;   // "typ1"
            
            bool isValidOTF = signature == 0x4F54544F; // "OTTO"
            
            if (!isValidTTF && !isValidOTF)
            {
                LogHelper.Debug($"字体文件头标识无效: {fontFilePath}, signature: 0x{signature:X8}");
                return false;
            }
        }
        else if (extension == ".ttc")
        {
            // TrueType Collection 字体文件头检查
            uint signature = (uint)((fontData[0] << 24) | (fontData[1] << 16) | 
                                   (fontData[2] << 8) | fontData[3]);
            
            if (signature != 0x74746366) // "ttcf"
            {
                LogHelper.Debug($"TTC 字体文件头标识无效: {fontFilePath}, signature: 0x{signature:X8}");
                return false;
            }
        }

        return true;
    }
    catch (Exception ex)
    {
        LogHelper.Debug($"字体文件验证异常: {fontFilePath} - {ex.Message}");
        return false;
    }
}
```

**验证内容：**

1. **文件大小检查**
   - 最小 12 字节（字体文件头）
   - 过滤掉损坏的文件

2. **TrueType/OpenType 文件头**
   - `0x00010000` - TrueType 标准标识
   - `0x74727565` - "true" (Mac TrueType)
   - `0x74797031` - "typ1" (Type 1)
   - `0x4F54544F` - "OTTO" (OpenType)

3. **TrueType Collection 文件头**
   - `0x74746366` - "ttcf" (TTC 标识)

#### B. 在 GetFont 方法中应用验证

```csharp
// 4. 读取字体文件
```

- 提供更清晰的错误信息

## 技术细节

### 字体文件格式

#### TrueType Font (.ttf)
```
偏移  大小  说明
0     4     sfnt version (0x00010000 或 "true")
4     2     numTables
6     2     searchRange
8     2     entrySelector
10    2     rangeShift
```

#### OpenType Font (.otf)
```
偏移  大小  说明
0     4     sfnt version ("OTTO")
4     2     numTables
...
```

#### TrueType Collection (.ttc)
```
偏移  大小  说明
0     4     TTC Tag ("ttcf")
4     4     Version
8     4     numFonts
...
```

### 异常处理层次

```
1. SystemFontResolver.GetFont()
   ├─ 文件读取异常
   ├─ 文件验证异常
   └─ 返回 null（不抛出异常）

2. TestPdfSharpFontRendering()
   ├─ NullReferenceException（PDFsharp 内部）
   ├─ ArgumentException（参数错误）
   ├─ 其他异常
   └─ 返回 false（不抛出异常）

3. 异步测试任务
   ├─ 捕获测试异常
   ├─ 捕获 UI 更新异常
   └─ 记录日志（不传播异常）

4. UpdateFontPreview()
   ├─ 捕获所有异常
   ├─ 回退到默认字体
   └─ 记录日志
```

## 常见问题

### Q1: 为什么有些字体显示不兼容？

**A:** 可能的原因：

1. **字体文件格式问题**
   - 不完全符合 OpenType 规范
   - 缺少必要的字体表
   - 文件损坏

2. **PDFsharp 限制**
   - PDFsharp 6.2.0 对某些格式支持有限
   - 某些特殊字体无法解析

3. **字体保护**
   - 有 DRM 保护
   - 需要特殊授权

**解决方法：**
- 选择常用字体（微软雅黑、宋体、Arial 等）
- 这些字体经过充分测试，兼容性最好

### Q2: 字体验证失败会影响程序吗？

**A:** 不会。

- 验证失败只是标记为"不兼容"
- 程序继续正常运行
- 用户可以选择其他字体

### Q3: 如何查看详细的错误信息？

**A:** 查看日志文件。

**日志位置：**
```
%AppData%\Roaming\大诚重命名工具\Logs\log_YYYYMMDD.txt
```

**关键日志：**
- `PDFsharp 字体解析失败` - 字体格式问题
- `字体文件验证失败` - 文件头检查失败
- `字体测试异常` - 测试过程异常
