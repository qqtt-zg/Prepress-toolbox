# 转曲按钮调试步骤

## 问题描述
转曲按钮显示为灰色，无法点击

## 调试步骤

### 1. 运行应用程序
```
.\src\WindowsFormsApp3\bin\Debug\net48\win-x64\大诚重命名工具.exe
```

### 2. 打开PDF文件
- 点击"PDF操作"面板
- 点击"打开"按钮
- 选择任意PDF文件

### 3. 打开检查器
- 点击"检查器"按钮
- 检查器窗口应该打开

### 4. 切换到字体标签页
- 在检查器窗口中，点击"字体"标签页
- 应该能看到字体列表和顶部的"转曲"按钮

### 5. 检查按钮状态
- 查看"转曲"按钮是否可点击（不是灰色）
- 如果仍然是灰色，查看日志文件

### 6. 查看日志
查看最新的日志文件，搜索以下关键字：
- `[PdfInspectorControl] 转曲按钮已创建`
- `[PdfInspectorControl] LoadPdf 开始`
- `[PdfInspectorControl] 启用转曲按钮`
- `[PdfInspectorControl] 转曲按钮已启用`

## 预期日志输出

```
[DEBUG] [PdfInspectorControl] 转曲按钮已创建: Enabled=False, Visible=True
[DEBUG] [PdfInspectorControl] LoadPdf 开始: C:\path\to\file.pdf, 页码: 1
[DEBUG] [PdfInspectorControl] _outlineButton 是否为 null: False
[DEBUG] [PdfInspectorControl] PDF信息加载成功，开始更新显示
[DEBUG] [PdfInspectorControl] 启用转曲按钮，当前状态: Enabled=False, Visible=True
[DEBUG] [PdfInspectorControl] 转曲按钮已启用: Enabled=True
```

## 可能的问题

1. **LoadPdf 未被调用**
   - 检查 `PdfInspectorForm.LoadPdf` 是否被调用
   - 检查 `PdfInspectorControl.LoadPdf` 是否被调用

2. **LoadPdf 抛出异常**
   - 查看日志中是否有错误信息
   - 检查 PDF 文件是否损坏

3. **按钮未正确创建**
   - 查看日志中是否有"转曲按钮已创建"消息
   - 检查 `_outlineButton` 是否为 null

4. **UI 线程问题**
   - 按钮可能在非 UI 线程上被修改
   - 需要使用 Invoke 来更新 UI

## 下一步

根据日志输出确定问题所在，然后采取相应的修复措施。
