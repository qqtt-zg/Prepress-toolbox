# 系统字体兼容性提升方案

## 问题分析

### 原始问题
大部分系统字体在 PDFsharp 6.2.0 中显示为不兼容。

### 根本原因

1. **PDFsharp 6.2.0 默认行为**
   - Core 构建默认不支持系统字体
   - 需要手动实现 `IFontResolver` 接口
   - 直接使用字体名称创建 `XFont` 会失败

2. **字体文件访问问题**
   - PDFsharp 无法自动找到系统字体文件
   - 需要直接从 `C:\Windows\Fonts` 读取字体文件
   - 需要将字体文件字节数据提供给 PDFsharp

3. **字体名称映射问题**
   - 显示名称（如"微软雅黑"）与文件名（如"msyh.ttf"）不一致
   - 需要建立映射关系

## 解决方案

### 1. 自定义字体解析器（SystemFontResolver.cs）

实现 `IFontResolver` 接口，直接从系统字体文件夹加载字体。

#### 核心功能

**A. 字体文件扫描**
```csharp
private void LoadFontFiles()
{
    // 扫描 C:\Windows\Fonts 目录
    var fontFiles = Directory.GetFiles(WindowsFontsPath, "*.ttf")
        .Concat(Directory.GetFiles(WindowsFontsPath, "*.ttc"))
        .Concat(Directory.GetFiles(WindowsFontsPath, "*.otf"))
        .ToList();
    
    // 建立文件名到路径的映射
    foreach (var fontFile in fontFiles)
    {
        string fileName = Path.GetFileNameWithoutExtension(fontFile).ToLower();
        _fontFileCache[fileName] = fontFile;
    }
}
```

**B. 字体名称映射**
```csharp
private static readonly Dictionary<string, string> FontNameMapping = new Dictionary<string, string>
{
    // 中文字体
    { "Microsoft YaHei", "msyh" },
    { "微软雅黑", "msyh" },
    { "SimSun", "simsun" },
    { "宋体", "simsun" },
    { "SimHei", "simhei" },
    { "黑体", "simhei" },
    
    // 英文字体
    { "Arial", "arial" },
    { "Times New Roman", "times" },
    { "Courier New", "cour" },
    // ... 更多映射
};
```

**C. 字体解析**
```csharp
public FontResolverInfo ResolveTypeface(string familyName, bool bold, bool italic)
{
    // 1. 从映射表查找基础字体名
    string baseFontName = FontNameMapping.TryGetValue(familyName, out string mapped) 
        ? mapped 
        : familyName.ToLower().Replace(" ", "");
    
    // 2. 根据样式构建文件名
    string faceName = BuildFaceName(baseFontName, bold, italic);
    
    // 3. 查找字体文件
    string fontFilePath = FindFontFile(faceName, baseFontName, bold, italic);
    
    if (fontFilePath != null)
    {
        return new FontResolverInfo(faceName, false, false);
    }
    
    return null;
}
```

**D. 字体数据加载**
```csharp
public byte[] GetFont(string faceName)
{
    // 1. 检查缓存
    if (_fontDataCache.TryGetValue(faceName, out byte[] cachedData))
    {
        return cachedData;
    }
    
    // 2. 查找字体文件
    string fontFilePath = FindFontFile(...);
    
    // 3. 读取字体文件字节数据
    byte[] fontData = File.ReadAllBytes(fontFilePath);
    
    // 4. 缓存数据
    _fontDataCache[faceName] = fontData;
    
    return fontData;
}
```

#### 字体样式处理

**样式后缀规则：**
- Regular: 无后缀
- Bold: `_b` 或 `bd` 或 `b`
- Italic: `_i` 或 `i`
- Bold Italic: `_bi` 或 `bi` 或 `z`

**查找优先级：**
```csharp
// 对于 Arial Bold，尝试以下文件名：
1. arialbd.ttf    // 精确匹配
2. arialb.ttf     // 简写
3. arial_bold.ttf // 描述性
4. arial.ttf      // 回退到 Regular
```

### 2. 字体初始化器（PdfSharpFontInitializer.cs）

在应用程序启动时设置全局字体解析器。

```csharp
public static void Initialize()
{
    lock (_lock)
    {
        if (_initialized)
        {
            return;
        }

        // 创建自定义字体解析器
        var fontResolver = new SystemFontResolver();

        // 设置全局字体解析器
        GlobalFontSettings.FontResolver = fontResolver;

        _initialized = true;
    }
}
```

**关键点：**
- 必须在创建任何 `XFont` 对象之前调用
- 只能设置一次
- 线程安全

### 3. 程序启动集成（Program.cs）

在 `Main` 方法中添加初始化代码：

```csharp
// 初始化 PDFsharp 字体解析器（提高系统字体兼容性）
try
{
    PdfSharpFontInitializer.Initialize();
    LogHelper.Info("PDFsharp 字体解析器初始化完成");
}
catch (Exception ex)
{
    LogHelper.Error($"PDFsharp 字体解析器初始化失败: {ex.Message}", ex);
}
```

### 字体缓存机制

**两级缓存：**

1. **文件路径缓存**
   ```csharp
   Dictionary<string, string> _fontFileCache
   // 键：字体文件名（不含扩展名）
   // 值：完整文件路径
   ```

2. **字体数据缓存**
   ```csharp
   Dictionary<string, byte[]> _fontDataCache
   // 键：faceName
   // 值：字体文件字节数据
   ```

**优势：**
- 避免重复扫描文件系统
- 避免重复读取字体文件

### 不兼容的字体类型

即使使用自定义解析器，以下字体可能仍然不兼容：

1. **特殊格式字体**
   - 某些 PostScript 字体
   - 某些位图字体
   - 某些矢量字体

2. **损坏的字体文件**
   - 文件损坏
   - 格式错误

3. **受保护的字体**
   - 有 DRM 保护的字体
   - 需要特殊授权的字体

4. **不完整的字体**
   - 缺少必要的字体表
   - 字体元数据不完整

## 故障排除

### 问题 1: 字体仍然不兼容

**可能原因：**
- 字体文件不存在
- 字体文件损坏
- 字体格式不支持

**解决方法：**
1. 检查日志，查看具体错误
2. 尝试重新安装字体
3. 选择其他字体

### 问题 2: 初始化失败

**可能原因：**
- 无法访问 `C:\Windows\Fonts` 目录
- 权限不足

**解决方法：**
1. 以管理员身份运行
2. 检查文件系统权限
3. 查看详细日志

### 问题 3: 字体显示异常

**可能原因：**
- 字体文件版本问题
- 字体样式不匹配

**解决方法：**
1. 使用 Regular 样式
2. 更新字体文件
3. 选择其他字体

## 性能优化

### 缓存策略

1. **启动时扫描**
   - 一次性扫描所有字体文件
   - 建立文件名到路径的映射
   - 时间：约 100-200ms

2. **延迟加载**
   - 只在需要时读取字体文件
   - 读取后缓存字节数据
   - 避免重复 I/O

3. **内存管理**
   - 字体数据缓存在内存中
   - 典型字体文件：5-20MB
   - 10 个字体约占用：50-200MB

### 性能指标

| 操作 | 时间 | 说明 |
|------|------|------|
| 初始化解析器 | 100-200ms | 启动时一次 |
| 解析字体名称 | <1ms | 查找映射表 |
| 首次加载字体 | 10-50ms | 读取文件 |
| 缓存命中 | <1ms | 从内存读取 |
| 字体兼容性测试 | 100-200ms | 创建 PDF 测试 |

## 扩展性

### 添加新字体映射

在 `SystemFontResolver.cs` 中添加：

```csharp
private static readonly Dictionary<string, string> FontNameMapping = new Dictionary<string, string>
{
    // 现有映射...

```

### 支持自定义字体目录

可以扩展 `LoadFontFiles` 方法：

```csharp
private void LoadFontFiles()
{
    // 系统字体目录
    ScanDirectory(WindowsFontsPath);
    
    // 自定义字体目录
    string customFontsPath = Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "Fonts");
    if (Directory.Exists(customFontsPath))
    {
        ScanDirectory(customFontsPath);
    }
}
```

### 支持字体回退

可以实现 `FallbackFontResolver`：

```csharp
GlobalFontSettings.FallbackFontResolver = new FallbackFontResolver();
```
