# 修改记录 - 移除确认对话框和成功提示

## 修改时间
2026-01-19

## 修改原因

现在已经有了完善的历史栈系统，支持多级撤回/重做，用户可以随时撤回不满意的操作。因此不再需要在每次操作时弹出确认对话框和成功提示，这样可以：

1. **提升操作流畅性** - 减少不必要的点击
2. **符合专业软件习惯** - Adobe Acrobat DC、Photoshop 等专业软件都不会在撤回/重做时弹框
3. **降低操作成本** - 用户可以快速尝试和撤回
4. **提高工作效率** - 减少中断，保持工作流

## 修改内容

### 1. 移除撤回操作的警告和成功提示

#### 之前
```csharp
private void BtnUndo_Click(object sender, EventArgs e)
{
    if (_undoStack.Count == 0)
    {
        ShowWarning("没有可以撤回的操作"); // ❌ 移除警告
        return;
    }
    
    // ... 撤回逻辑 ...
    
    ShowSuccess("已撤回操作"); // ❌ 移除成功提示
}
```

#### 现在
```csharp
private void BtnUndo_Click(object sender, EventArgs e)
{
    if (_undoStack.Count == 0)
    {
        return; // ✅ 按钮应该是禁用的，静默返回
    }
    
    // ... 撤回逻辑 ...
    
    // ✅ 不显示成功提示，只更新状态栏
}
```

## 保留的提示

### 仍然保留的错误提示
以下错误提示仍然保留，因为它们是真正的错误情况：

1. **转曲失败** - 显示错误通知
```csharp
AntdUI.Notification.error(this.FindForm(), "转曲失败", 
    "字体转曲失败，请查看日志了解详情");
```

2. **撤回/重做失败** - 显示错误消息
```csharp
ShowError($"撤回失败: {ex.Message}");
ShowError($"重做失败: {ex.Message}");
```

3. **文件加载失败** - 显示错误消息
```csharp
AntdUI.Notification.warn(this.FindForm(), "提示", "请先加载PDF文件");
```

### 状态栏信息
所有操作仍然会更新状态栏，提供即时反馈：

- 转曲后: "已转曲 (未保存) - 文件名.pdf"
- 撤回后: "已撤回 (未保存) - 文件名.pdf" 或 "已加载: 文件名.pdf"
- 重做后: "已重做 (未保存) - 文件名.pdf"
- 保存后: "已保存: 文件名.pdf"

## 安全性保障

虽然移除了确认对话框，但系统仍然很安全：

1. **多级撤回** - 可以撤回最多10次操作
2. **多级重做** - 可以重做被撤回的操作
3. **未保存提示** - 状态栏清晰显示"(未保存)"
4. **保存按钮** - 有更改时保存按钮启用
5. **另存为** - 可以保存副本而不覆盖原文件
