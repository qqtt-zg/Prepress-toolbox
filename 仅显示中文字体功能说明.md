## 功能说明

### 修改内容

将字体列表从"显示所有系统字体"改为"仅显示与 iTextSharp.LGPLv2 兼容的字体"。

### 修改原因

- 用户需要使用 PDF 兼容的字体来生成标识页
- 过滤掉不兼容的字体可以避免生成失败
- 提升用户体验，确保选择的字体都能正常工作

### 版本历史

**版本 1.0（已弃用）：**
- 只显示支持中文的字体
- 使用 WinForms `MeasureString` 检测中文支持
- 问题：部分支持中文的字体不兼容 PDF

**版本 2.0（当前）：**
- 只显示 PDF 兼容的字体
- 使用 iTextSharp.LGPLv2 实际测试兼容性
- 优势：确保所有字体都能在 PDF 中正常使用

## 技术实现

### 1. PDF 兼容性检测方法

**实现位置：** `SettingsFontTextControl.cs`

```csharp
/// <summary>
/// 检查字体是否与 iTextSharp.LGPLv2 兼容
/// </summary>
private bool IsPdfCompatible(string fontName)
{
    try
    {
        // 使用 iTextSharp.LGPLv2.Core 测试字体兼容性
        return ITextSharpLGPLFontHelper.TestFontCompatibility(fontName);
    }
    catch
    {
        return false;
    }
}
```

**检测原理：**
1. 调用 `ITextSharpLGPLFontHelper.TestFontCompatibility()`
2. 该方法尝试创建 `BaseFont` 对象
3. 测试文本宽度计算
4. 如果成功，返回 `true`（兼容）
5. 如果失败，返回 `false`（不兼容）

**底层实现：** `ITextSharpLGPLFontHelper.cs`

```csharp
public static bool TestFontCompatibility(string fontName)
{
    try
    {
        // 1. 创建 BaseFont
        BaseFont baseFont = CreateBaseFont(fontName);
        
        if (baseFont == null)
        {
            return false;
        }
        
        // 2. 测试文本宽度
        float width = baseFont.GetWidthPoint("Test", 12);
        
        return width > 0;
    }
    catch
    {
        return false;
    }
}
```

**优势：**
- ✅ 使用实际的 PDF 库进行测试
- ✅ 确保字体能在 PDF 中正常使用
- ✅ 支持 TTF、TTC、OTF 等多种格式
- ✅ 自动处理 TTC 索引

### 2. 字体加载流程修改

**修改前（版本 1.0）：**
```csharp
// 添加支持中文的字体到列表
foreach (FontFamily font in sortedFonts)
{
    try
    {
        // 验证字体是否支持中文
        if (SupportsChinese(font))
        {
            allFonts.Add(font.Name);
        }
    }
    catch { }
}
```

**修改后（版本 2.0）：**
```csharp
// 添加PDF兼容的字体到列表
int processedCount = 0;
foreach (FontFamily font in sortedFonts)
{
    processedCount++;
    
    try
    {
        // 验证字体是否与 iTextSharp.LGPLv2 兼容
        if (IsPdfCompatible(font.Name))
        {
            allFonts.Add(font.Name);
            LogHelper.Debug($"加载PDF兼容字体: {font.Name}");
        }
        else
        {
            LogHelper.Debug($"字体 {font.Name} 不兼容PDF，已跳过");
        }
    }
    catch (Exception ex)
    {
        LogHelper.Debug($"字体 {font.Name} 验证失败: {ex.Message}");
    }
    
    // 更新进度
    if (processedCount % 10 == 0 || processedCount == totalCount)
    {
        UpdateLoadingProgress(processedCount, totalCount, allFonts.Count);
    }
}
```

## 技术细节

### PDF 兼容性检测原理

**为什么使用 iTextSharp.LGPLv2 测试？**
- iTextSharp.LGPLv2 是实际使用的 PDF 库
- 直接测试可以确保字体在 PDF 中可用
- 避免"WinForms 可用但 PDF 不可用"的问题

**检测流程：**
1. 查找系统字体文件（TTF、TTC、OTF）
2. 尝试创建 `BaseFont` 对象
3. 测试文本宽度计算
4. 如果成功，字体兼容
5. 如果失败，字体不兼容

**TTC 字体处理：**
```csharp
// 自动检测 TTC 并添加索引
if (extension == ".ttc")
{
    fontPath = fontPath + ",0"; // 0 = Regular
}

BaseFont baseFont = BaseFont.CreateFont(
    fontPath,
    BaseFont.IDENTITY_H,  // Unicode 支持
    BaseFont.EMBEDDED);   // 嵌入字体
```

### 性能优化建议

**当前实现：**
- 每次启动都重新扫描所有字体
- 加载时间 10-15 秒

**优化方案（未来）：**
1. **字体缓存** - 缓存兼容字体列表到文件
2. **增量更新** - 只检测新安装的字体
3. **并行处理** - 使用多线程并行测试（需要注意线程安全）
4. **后台预加载** - 应用启动时后台加载字体列表
