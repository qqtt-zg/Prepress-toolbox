# 修复记录：数据匹配正则表达式来源错误

## 问题描述

当前在导入表格，没有触发保留分组功能时，数据匹配中使用的正则表达式没有使用 `ExcelImportForm.cs` 的 `cmbRegex2` 控件选择的正则表达式，而错误使用了 `FileRenamePanel.cs` 的 `_cmbRegex` 控件选择的正则表达式来提取文件名中的正则结果进行数据匹配。

## 修复方案

采用以下方案修复问题：

1. 在 `IExcelImportService` 接口中添加 `SelectedRegexPattern` 属性
2. 在 `ExcelImportHelper` 类中实现该属性
3. 在 `ExcelImportForm.BtnOK_Click` 中保存选择的正则表达式到 `ExcelImportService`
4. 在 `FileRenamePanelPresenter.MatchExcelData` 中使用 `ExcelImportService.SelectedRegexPattern` 重新提取正则结果进行数据匹配

### 3. src/WindowsFormsApp3/Presenters/FileRenamePanelPresenter.cs

**修改内容 1**：添加 `ExtractRegexResultForMatching` 辅助方法

```csharp
/// <summary>
/// 从文件名中提取正则匹配结果（用于数据匹配）
/// 使用 ExcelImportService 中保存的正则表达式
/// </summary>
/// <param name="fileName">文件名</param>
/// <returns>正则匹配结果</returns>
private string ExtractRegexResultForMatching(string fileName)
{
    try
    {
        // ✅ 修复：使用 ExcelImportService 的正则表达式
        var pattern = _excelImportService?.SelectedRegexPattern;
        _logger?.LogInformation($"[ExtractRegexResultForMatching] 开始提取正则结果 - 文件名: {fileName}, 正则: {pattern}");
        
        if (string.IsNullOrEmpty(pattern))
        {
            _logger?.LogWarning($"[ExtractRegexResultForMatching] ExcelImportService 的正则表达式为空");
            return null;
        }

        var match = Regex.Match(fileName, pattern);
        if (match.Success)
        {
            // 优先使用第一个捕获组的值，如果没有捕获组则使用整个匹配
            string result;
            if (match.Groups.Count > 1 && match.Groups[1].Success)
            {
                result = match.Groups[1].Value;
                _logger?.LogInformation($"[ExtractRegexResultForMatching] 提取捕获组结果: '{result}'");
            }
            else
            {
                result = match.Value;
                _logger?.LogInformation($"[ExtractRegexResultForMatching] 使用完整匹配结果: '{result}'");
            }
            return result;
        }

        _logger?.LogWarning($"[ExtractRegexResultForMatching] 正则匹配失败 - 文件名: {fileName}, 正则: {pattern}");
        return null;
    }
    catch (Exception ex)
    {
        _logger?.LogError(ex, $"[ExtractRegexResultForMatching] 提取正则结果失败: {fileName}");
        return null;
    }
}
```

**位置**：在 `ExtractRegexResult` 方法之后

**修改内容 2**：修改 `MatchExcelData` 方法

```csharp
int matchedCount = 0;

foreach (var fileInfo in fileList)
{
    // ✅ 修复：使用 ExcelImportService 的正则表达式重新提取正则结果
    string regexResultForMatching = ExtractRegexResultForMatching(fileInfo.OriginalName ?? "");
    
    if (string.IsNullOrEmpty(regexResultForMatching))
    {
        _logger?.LogDebug($"跳过文件（无正则结果）: {fileInfo.OriginalName}");
        continue;
    }

    // ✅ 新增: 检测并使用保留的正则结果
    // 检查是否为返单文件(包含保留分组前缀)
    if (!string.IsNullOrEmpty(fileInfo.OriginalName))
    {
        string originalNameWithoutExt = Path.GetFileNameWithoutExtension(fileInfo.OriginalName);
        
        // 检测是否包含保留分组前缀模式 (如 &ID-, &MT- 等)
        if (System.Text.RegularExpressions.Regex.IsMatch(originalNameWithoutExt, @"&[A-Z]+-"))
        {
            _logger?.LogDebug($"检测到保留分组前缀，尝试提取保留的正则结果: '{originalNameWithoutExt}'");
            
            // 获取保留分组配置
            var preserveGroupConfig = GetPreserveGroupConfig();
            
            if (preserveGroupConfig != null && preserveGroupConfig.Count > 0)
            {
                // 获取当前的前缀配置
                var currentPrefixes = GetCurrentPrefixes();
                
                // 创建临时的 FileNameComponents 对象用于提取保留数据
                var tempComponents = new Models.FileNameComponents
                {
                    OriginalFileName = originalNameWithoutExt,
                    PreserveGroupConfig = preserveGroupConfig,
                    Prefixes = currentPrefixes
                };
                
                // 提取保留的分组数据
                var preserveData = tempComponents.ExtractPreserveGroupData(originalNameWithoutExt);
                
                // 如果保留数据中包含正则结果，则使用保留的值
                if (preserveData != null && preserveData.ContainsKey("正则结果"))
                {
                    string preservedRegexResult = preserveData["正则结果"];
                    if (!string.IsNullOrEmpty(preservedRegexResult))
                    {
                        regexResultForMatching = preservedRegexResult;
                        _logger?.LogInformation($"✅ 使用保留的正则结果进行匹配: '{regexResultForMatching}' (重新提取的结果: '{regexResultForMatching}', 文件: '{fileInfo.OriginalName}')");
                    }
                }
                else
                {
                    _logger?.LogDebug($"保留数据中未找到正则结果，使用重新提取的RegexResult: '{regexResultForMatching}'");
                }
            }
            else
            {
                _logger?.LogDebug($"保留分组配置为空，使用重新提取的RegexResult: '{regexResultForMatching}'");
            }
        }
    }

    var matchData = MatchExcelData(regexResultForMatching);
    // ... 后续处理
}
```

## 修复效果

修复后，系统将：

1. **正确使用 ExcelImportForm 的正则表达式**：
   - 在数据匹配时，使用 `ExcelImportForm.cmbRegex2` 控件选择的正则表达式
   - 该正则表达式存储在 `ExcelImportService.SelectedRegexPattern` 属性中

2. **保持 FileRenamePanel 的正则表达式独立**：
   - 文件名处理时，继续使用 `FileRenamePanel._cmbRegex` 控件选择的正则表达式
   - 两者互不干扰，各司其职

3. **支持保留分组功能**：
   - 如果文件包含保留分组前缀，优先使用保留的正则结果
   - 否则使用 `ExcelImportService.SelectedRegexPattern` 重新提取正则结果

4. **增强日志记录**：
   - 记录使用的正则表达式来源
   - 记录提取的正则结果
   - 便于调试和问题排查

## 注意事项

1. **向后兼容性**：
   - 如果 `ExcelImportService.SelectedRegexPattern` 为空，`ExtractRegexResultForMatching` 将返回 `null`
   - 系统会跳过该文件的数据匹配，不会抛出异常

2. **保留分组优先级**：
   - 保留的正则结果优先级最高
   - 如果文件包含保留分组前缀且保留数据中有正则结果，则使用保留的值
   - 否则使用 `ExcelImportService.SelectedRegexPattern` 重新提取

3. **日志级别**：
   - 使用 `LogInformation` 记录关键操作
   - 使用 `LogDebug` 记录详细信息
   - 使用 `LogWarning` 记录异常情况
   - 使用 `LogError` 记录错误

4. **性能影响**：
   - 每次数据匹配时都会重新提取正则结果
   - 对于大量文件，可能会有轻微的性能影响
   - 但这是必要的，以确保使用正确的正则表达式
