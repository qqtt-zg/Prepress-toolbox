# 快速参考：数据匹配正则表达式修复

## 问题概述

**问题**：数据匹配时错误使用了 `FileRenamePanel` 的正则表达式，而不是 `ExcelImportForm` 的正则表达式。

**影响**：导致数据匹配结果不符合预期。

**修复状态**：✅ 已修复

---

## 修复内容

### 1. 新增属性

在 `IExcelImportService` 接口和 `ExcelImportHelper` 类中添加：

```csharp
string SelectedRegexPattern { get; set; }
```

### 2. 保存正则表达式

在 `ExcelImportHelper.ShowImportForm` 方法中：

```csharp
_selectedRegexPattern = _excelImportFormInstance.SelectedRegexPattern;
```

### 3. 新增方法

在 `FileRenamePanelPresenter` 类中添加：

```csharp
private string ExtractRegexResultForMatching(string fileName)
{
    var pattern = _excelImportService?.SelectedRegexPattern;
    // ... 使用 ExcelImportService 的正则表达式提取结果
}
```

### 4. 修改数据匹配逻辑

在 `FileRenamePanelPresenter.MatchExcelData` 方法中：

```csharp
// 使用 ExcelImportService 的正则表达式重新提取正则结果
string regexResultForMatching = ExtractRegexResultForMatching(fileInfo.OriginalName ?? "");
```

## 使用说明

### 正常使用流程

1. **导入 Excel 数据**：
   - 点击"导入 Excel"按钮
   - 在 `ExcelImportForm` 中选择正则表达式
   - 配置搜索列、数量列等
   - 点击"确定"

2. **数据匹配**：
   - 系统自动使用 `ExcelImportForm` 选择的正则表达式
   - 从文件名中提取正则结果
   - 在 Excel 数据中查找匹配的行
   - 更新文件的数量、序号、材料等信息

3. **文件名处理**：
   - 在 `FileRenamePanel` 中选择正则表达式
   - 系统使用该正则表达式处理文件名
   - 不影响数据匹配结果

### 保留分组场景

如果文件包含保留分组前缀（如 `&ID-123&MT-PVC-`），系统会：

1. 检测保留分组前缀
2. 提取保留的正则结果
3. 优先使用保留的正则结果进行数据匹配
4. 如果保留数据中没有正则结果，则使用 `ExcelImportService.SelectedRegexPattern` 重新提取

## 常见问题

### Q1：数据匹配结果为空？

**可能原因**：
1. `ExcelImportForm` 中未选择正则表达式
2. 正则表达式无法匹配文件名
3. Excel 数据中没有匹配的行

**解决方法**：
1. 检查 `ExcelImportForm` 中是否选择了正则表达式
2. 检查正则表达式是否正确
3. 检查 Excel 数据中是否有匹配的行

### Q2：数据匹配使用了错误的正则表达式？

**可能原因**：
1. 修复未生效
2. 缓存问题

**解决方法**：
1. 重新编译应用程序
2. 清空 Excel 数据后重新导入
3. 检查日志中的正则表达式

### Q3：保留分组场景下数据匹配失败？

**可能原因**：
1. 保留分组配置错误
2. 保留数据中没有正则结果

**解决方法**：
1. 检查保留分组配置
2. 检查日志中的保留数据提取过程
3. 确认文件名包含正确的保留分组前缀
